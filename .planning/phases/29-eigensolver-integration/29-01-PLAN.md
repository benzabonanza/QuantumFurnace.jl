Create the new file with the following components (in order):

**2a. `KrylovGapResult{T}` struct:**
```julia
struct KrylovGapResult{T<:AbstractFloat}
    eigenvalues::Vector{Complex{T}}       # Lindbladian eigenvalues (sorted by |Re(lambda)|)
    spectral_gap::T                       # abs(real(eigenvalues[2]))
    fixed_point::Matrix{Complex{T}}       # Steady-state density matrix (Hermitianized, trace-normalized)
    gap_mode::Matrix{Complex{T}}          # Gap mode operator (eigvec 2, reshaped dim x dim)
    converged::Int                        # Number of converged eigenvalues
    matvec_count::Int                     # info.numops
    num_restarts::Int                     # info.numiter
    normres::Vector{T}                    # Residual norms
    channel_eigenvalues::Union{Nothing, Vector{Complex{T}}}  # Raw channel eigenvalues (nothing for Lindbladian path)
    delta_used::Union{Nothing, T}         # Delta from ThermalizeConfig (nothing for Lindbladian path)
end
```

**2b. `_check_krylov_memory(n_qubits, krylovdim)` function:**

Computes `estimated_bytes = krylovdim * (4^n_qubits) * 16 * 1.5`. Gets `available = Sys.free_memory()`. If `estimated_bytes > 0.8 * available`, issues `@warn` with the estimate and available memory in GB. Returns nothing (advisory only, not a hard error). Per locked decision.

**2c. `_eigsolve_with_retry(f, x0, howmany, which; krylovdim, tol, maxiter, max_retries)` function:**

Retry wrapper per research recommendation:
- Use KrylovKit `Arnoldi(; krylovdim=current_krylovdim, tol=tol, maxiter=maxiter, verbosity=0)` algorithm
- Loop up to `max_retries + 1` attempts
- On each retry, multiply krylovdim by 1.5 (ceil to Int)
- Issue `@warn` on each retry with converged count and new krylovdim
- On final failure, throw error: "KrylovKit failed to converge: N/howmany eigenvalues after M attempts (final krylovdim=K)"
- Default: krylovdim=30, tol=1e-10, maxiter=100, max_retries=3

**2d. `_thermalize_to_liouv_config(tc)` functions:**

Two methods:
- `_thermalize_to_liouv_config(tc::ThermalizeConfig)` -> builds a `LiouvConfig` copying all shared fields (num_qubits, with_coherent, with_linear_combination, domain, beta, sigma, gaussian_parameters, a, b, num_energy_bits, t0, w0, eta, num_trotter_steps_per_t0). Omits mixing_time and delta.
- `_thermalize_to_liouv_config(tc::ThermalizeConfigGNS)` -> builds a `LiouvConfigGNS` similarly. Note: with_coherent must be false for GNS.

**2e. `apply_delta_channel!(ws, rho, delta, config_liouv, hamiltonian)` function:**

Compute E(rho) = rho + delta * L(rho):
1. Call `apply_lindbladian!(ws, rho, config_liouv, hamiltonian)` which writes L(rho) to `ws.rho_out`
2. Then `@. ws.rho_out = rho + delta * ws.rho_out`
3. Return `ws.rho_out`

Type signature: `where {T<:Complex}` on `ws::KrylovWorkspace{T}`, `rho::Matrix{T}`, `config_liouv::AbstractLiouvConfig`, `hamiltonian::HamHam`. `delta::Real`.

**2f. `krylov_spectral_gap` for Lindbladian path (`AbstractLiouvConfig`):**

```julia
function krylov_spectral_gap(
    config::AbstractLiouvConfig,
    hamiltonian::HamHam,
    jumps::Vector{JumpOp};
    trotter::Union{TrottTrott, Nothing}=nothing,
    krylovdim::Int=30,
    howmany::Int=4,
    tol::Real=1e-10,
    max_retries::Int=3,
    krylov_kwargs...
)
```

Implementation:
1. Guard: `krylovdim > howmany || error("krylovdim ($krylovdim) must be > howmany ($howmany)")`
2. Call `_check_krylov_memory(config.num_qubits, krylovdim)`
3. Allocate workspace: `ws = KrylovWorkspace(config, hamiltonian, jumps; trotter=trotter)`
4. Determine dim: `dim = size(hamiltonian.data, 1)`
5. Build matvec closure:
   ```julia
   function lindbladian_matvec(v::AbstractVector)
       rho = reshape(v, dim, dim)
       apply_lindbladian!(ws, rho, config, hamiltonian)
       return copy(vec(ws.rho_out))  # CRITICAL: copy to avoid aliasing (Pitfall 1)
   end
   ```
6. Initial vector: `x0 = vec(Matrix{ComplexF64}(I(dim) / dim))` (maximally mixed)
7. Call `_eigsolve_with_retry(lindbladian_matvec, x0, howmany, :LR; krylovdim, tol, max_retries, krylov_kwargs...)`
8. Sort eigenvalues by `abs(real(lambda))` ascending (steady state first, then gap mode)
9. Extract fixed_point from eigenvector[1]: `reshape(vecs[perm[1]], dim, dim)`, hermitianize!, trace-normalize
10. Extract gap_mode from eigenvector[2]: `reshape(vecs[perm[2]], dim, dim)`
11. Compute `spectral_gap = abs(real(eigenvalues_sorted[2]))`
12. Build and return `KrylovGapResult` with `channel_eigenvalues=nothing`, `delta_used=nothing`

**2g. `krylov_spectral_gap` for channel path (`AbstractThermalizeConfig`):**

```julia
function krylov_spectral_gap(
    config::AbstractThermalizeConfig,
    hamiltonian::HamHam,
    jumps::Vector{JumpOp};
    trotter::Union{TrottTrott, Nothing}=nothing,
    krylovdim::Int=30,
    howmany::Int=4,
    tol::Real=1e-10,
    max_retries::Int=3,
    krylov_kwargs...
)
```

Implementation:
1. Same guards as Lindbladian path
2. Convert config: `config_liouv = _thermalize_to_liouv_config(config)`
3. Get delta: `delta = config.delta`
4. Allocate workspace: `ws = KrylovWorkspace(config_liouv, hamiltonian, jumps; trotter=trotter)`
5. Build channel matvec closure:
   ```julia
   function channel_matvec(v::AbstractVector)
       rho = reshape(v, dim, dim)
       apply_delta_channel!(ws, rho, delta, config_liouv, hamiltonian)
       return copy(vec(ws.rho_out))  # CRITICAL: copy
   end
   ```
6. Initial vector: same maximally mixed state
7. Call `_eigsolve_with_retry(channel_matvec, x0, howmany, :LM; krylovdim, tol, max_retries, krylov_kwargs...)`
8. Store raw channel eigenvalues before conversion
9. Convert: `lindblad_eigenvalues = (channel_vals .- 1) ./ delta`
10. Sort Lindbladian eigenvalues by `abs(real(lambda))` ascending
11. Same fixed_point / gap_mode extraction and normalization as Lindbladian path (using the permutation from sorting)
12. Compute `spectral_gap = abs(real(lindblad_eigenvalues_sorted[2]))`
13. Build and return `KrylovGapResult` with `channel_eigenvalues=raw_channel_vals`, `delta_used=delta`

**CRITICAL implementation notes (from RESEARCH.md):**
- Always `copy(vec(ws.rho_out))` in the KrylovKit closure -- vec() returns a view that aliases ws.rho_out, which gets overwritten each call. Without copy, all Krylov basis vectors alias the same memory and eigsolve produces garbage.
- Use `Arnoldi` (not Lanczos) -- Lindbladian is non-Hermitian.
- The input `rho = reshape(v, dim, dim)` creates a view into KrylovKit's memory, which is safe because apply_lindbladian! reads from rho and writes to ws.rho_out (different buffer).
