  <action>
Create `test/test_krylov_eigsolve.jl` with the following testsets:

**Testset 1: "apply_delta_channel! round-trip vs dense"**

Test that `apply_delta_channel!` matches the dense formula `(I + delta*L_dense)*vec(rho)`:
1. Create `config = make_liouv_config(EnergyDomain(); with_coherent=true)`
2. Build dense Liouvillian: `L_dense = construct_lindbladian(TEST_JUMPS, config, TEST_HAM)`
3. Build workspace: `ws = KrylovWorkspace(config, TEST_HAM, TEST_JUMPS)`
4. Use `delta = 0.01`
5. For 5 random density matrices:
   - Compute `v_dense = (I + delta * L_dense) * vec(rho)` using `LinearAlgebra.I(DIM^2)`
   - Compute `apply_delta_channel!(ws, rho, delta, config, TEST_HAM)`
   - Compare `vec(ws.rho_out)` vs `v_dense` with `isapprox(; atol=1e-12)`

**Testset 2: "KrylovGapResult struct fields"**

Test that the struct has all expected fields and correct types:
1. Call `krylov_spectral_gap(config_kms, TEST_HAM, TEST_JUMPS; krylovdim=30, howmany=4)` with EnergyDomain KMS
2. Verify `result isa KrylovGapResult`
3. Check: `length(result.eigenvalues) >= 2`
4. Check: `result.spectral_gap > 0`
5. Check: `size(result.fixed_point) == (DIM, DIM)`
6. Check: `size(result.gap_mode) == (DIM, DIM)`
7. Check: `result.converged >= 2` (at least steady state + gap mode)
8. Check: `result.matvec_count > 0`
9. Check: `result.channel_eigenvalues === nothing` (Lindbladian path)
10. Check: `result.delta_used === nothing` (Lindbladian path)

**Testset 3: "Lindbladian eigsolve accuracy (EnergyDomain KMS)"**

Cross-validate Krylov spectral gap against dense reference:
1. Create `config = make_liouv_config(EnergyDomain(); with_coherent=true)`
2. Build dense Liouvillian and get reference: `L_dense = construct_lindbladian(TEST_JUMPS, config, TEST_HAM)`; use `extract_leading_eigendata(L_dense; n_modes=4)` for dense reference eigenvalues
3. Call `krylov_spectral_gap(config, TEST_HAM, TEST_JUMPS; krylovdim=30, howmany=4)`
4. Compare `result.spectral_gap` vs `dense_result.spectral_gap` with `isapprox(; rtol=1e-6)`
5. Verify fixed_point is close to Gibbs state: `trace_distance_h(result.fixed_point, TEST_GIBBS) < 1e-4` (Krylov fixed point may be less precise than dense)
6. Verify `abs(real(result.eigenvalues[1])) < 1e-8` (steady state near zero)

**Testset 4: "Lindbladian eigsolve accuracy (EnergyDomain GNS)"**

Same structure as testset 3 but with `make_liouv_config_gns(EnergyDomain())`. GNS Gibbs fixed point is approximate, so relax tolerance: compare Krylov gap vs dense gap with `rtol=1e-6`. Skip Gibbs trace distance check (GNS fixed point differs from exact Gibbs).

**Testset 5: "Channel eigsolve accuracy (ThermalizeConfig)"**

Test the CPTP channel path:
1. Create `config_therm = make_thermalize_config(EnergyDomain(); with_coherent=true, delta=0.01)`
2. Get dense reference from Lindbladian path: `config_liouv = make_liouv_config(EnergyDomain(); with_coherent=true)` and use `extract_leading_eigendata` on dense Liouvillian
3. Call `krylov_spectral_gap(config_therm, TEST_HAM, TEST_JUMPS; krylovdim=30, howmany=4)`
4. Compare `result.spectral_gap` vs `dense_result.spectral_gap` with `rtol=1e-3` (channel path has O(delta^2) error from the linear approximation)
5. Verify `result.channel_eigenvalues !== nothing`
6. Verify `result.delta_used == 0.01`
7. Verify channel eigenvalues are near 1: `abs(result.channel_eigenvalues[1]) ≈ 1` with `atol=0.01`

**Testset 6: "All domains work (Lindbladian path)"**

Verify krylov_spectral_gap succeeds for all 4 domains:
1. EnergyDomain: `make_liouv_config(EnergyDomain(); with_coherent=true)` -> verify `result.spectral_gap > 0`
2. TimeDomain: `make_liouv_config(TimeDomain(); with_coherent=true)` -> verify `result.spectral_gap > 0`
3. TrotterDomain: `make_liouv_config(TrotterDomain(); with_coherent=true)` -> verify `result.spectral_gap > 0`, pass `trotter=TEST_TROTTER` and use `TEST_TROTTER_JUMPS`
4. BohrDomain: `make_liouv_config(BohrDomain(); with_coherent=true)` -> verify `result.spectral_gap > 0`

Use `krylovdim=30, howmany=2` to keep test fast. Each domain test is a sub-testset.

**Testset 7: "Guard rails"**

1. Test krylovdim <= howmany guard: `@test_throws ErrorException krylov_spectral_gap(config, TEST_HAM, TEST_JUMPS; krylovdim=2, howmany=4)`
2. Test that memory guard runs without error (hard to test the @warn itself, just verify it doesn't throw): call with very large krylovdim on n=4 (should not warn since 4-qubit systems are small), verify it returns a result.

**Testset 8: "Eigenvalue sorting and conversion"**

Verify eigenvalue properties:
1. Lindbladian path: eigenvalues[1] should have smallest |Re(lambda)|, eigenvalues[2] should have second smallest. Verify `abs(real(result.eigenvalues[1])) <= abs(real(result.eigenvalues[2]))`
2. Channel path: verify `result.channel_eigenvalues` exists and the conversion formula gives the correct Lindbladian eigenvalues. Specifically: `result.eigenvalues ≈ (result.channel_eigenvalues .- 1) ./ result.delta_used` with appropriate sorting.

**Testing pattern notes:**
- Use `using Test, LinearAlgebra, Random, QuantumFurnace` at the top

- All tests run at n=4 (DIM=16) for speed
- The dense reference uses `extract_leading_eigendata` from diagnostics.jl (proven accurate in Phase 26)
- For testsets that need dense Liouvillian, use `construct_lindbladian(TEST_JUMPS, config, TEST_HAM)`
