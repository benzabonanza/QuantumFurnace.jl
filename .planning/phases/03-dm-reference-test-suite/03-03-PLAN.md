**1. Create test/test_aqua.jl:**

```julia
using Aqua
using QuantumFurnace

@testset "Aqua.jl Package Quality (TINF-03)" begin
    Aqua.test_all(QuantumFurnace;
        ambiguities = false,          # Disable: multiple dispatch may create legitimate ambiguities
        deps_compat = (ignore=[:Pkg, :Printf, :Random, :LinearAlgebra, :SparseArrays,
                               :SharedArrays, :Distributed, :Base],),
        stale_deps = (ignore=[:Pkg],),
        piracies = false,             # Disable: kron! on AbstractMatrix may be flagged
    )
end
```

**Known exclusions and rationale:**
- `ambiguities = false`: Package uses heavy multiple dispatch across domain types; some ambiguities are structurally inherent.
- `deps_compat ignore`: Standard library packages don't need compat bounds in Julia 1.x. `Pkg` is used directly in the module.
- `stale_deps ignore Pkg`: `Pkg` is used for `Pkg.project().path` in the module but Aqua may not detect indirect usage.
- `piracies = false`: Package defines `kron!` on `AbstractMatrix` which is a deliberate extension.

**3. If Aqua tests fail:** Read the error output carefully. Common fixes:
- If `deps_compat` fails on additional stdlib packages, add them to the ignore list.
- If `stale_deps` fails on additional packages, add them to the ignore list.
