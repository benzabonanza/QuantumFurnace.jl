1. **Add `_precompute_R` for BohrDomain in `trajectories.jl`** (after the existing Time/TrotterDomain variant, around line 300).

   New method signature:
   ```julia
   function _precompute_R(
       jumps::AbstractVector{<:JumpOp},
       hamiltonian::HamHam,
       config::Config{Thermalize, BohrDomain},
       precomputed_data,
       scratch::ThermalizeScratch{<:Complex},
   )
   ```

   Implementation: Extract ONLY the R-accumulation logic from the existing BohrDomain `_jump_contribution!` in `jump_workers.jl` (lines 226-282). The R accumulation is:
   - For each `nu_2` in `bohr_keys`: compute `scratch.jump_oft = alpha(hamiltonian.bohr_freqs, nu_2) * jump.in_eigenbasis`
   - Accumulate `scratch.R += gamma_norm_factor * A_{nu_2}^dagger * B_{nu_2}` using the sparse index-based inner loop
   - Use `precomputed_data.bohr_keys`, `precomputed_data.bohr_is`, `precomputed_data.bohr_js` when available (Thermalize BohrDomain precomputed_data has these)
   - If `bohr_is` not available, fall back to `hamiltonian.bohr_dict[nu_2]` indices

   The scaling factor for R: use `gamma_norm_factor` (bare, no 1/p_jump -- that scaling is applied separately by the caller, matching Energy/TimeDomain convention). This means inside the inner loop where the existing `_jump_contribution!` uses `jump_weight_scaling = gamma_norm_factor / jump_prob`, the `_precompute_R` version uses just `gamma_norm_factor`.

   End with `hermitianize!(scratch.R)` and `return scratch.R`.

   Reference the Krylov's `_accumulate_R_total!` for BohrDomain (krylov_workspace.jl lines 188-216) as a cross-check on the structure, but note that function sums across ALL jumps while this one takes a `jumps` vector (will be called with `[single_jump]`).

2. **Add `_precompute_per_jump_channels` in `furnace_utensils.jl`** (after `_build_cptp_channel`, around line 201).

   ```julia
   function _precompute_per_jump_channels(
       jumps::Vector{JumpOp},
       ham_or_trott::Union{HamHam, TrottTrott},
       config::Config{Thermalize},
       precomputed_data;
       rescale_by_inv_prob::Bool = true,
   )
   ```

   Implementation (mirror `_build_trajectory_workspace` lines 105-122 from trajectories.jl):
   - `CT = Complex{eltype(ham_or_trott isa HamHam ? ham_or_trott.eigvals : ham_or_trott.eigvals)}`
   - `dim = size(ham_or_trott isa HamHam ? ham_or_trott.data : ham_or_trott.data, 1)`
   - `n_jumps = length(jumps)`
   - `p_jump = 1.0 / n_jumps`
   - Allocate `K0s = Vector{Matrix{CT}}(undef, n_jumps)` and `U_residuals = Vector{Matrix{CT}}(undef, n_jumps)`
   - Create temporary `builder_scratch = ThermalizeScratch(CT, dim)`
   - For each jump `a in 1:n_jumps`:
     - Call `_precompute_R([jumps[a]], ham_or_trott, config, precomputed_data, builder_scratch)`
     - `R_a = copy(builder_scratch.R)`
     - If `rescale_by_inv_prob`: `R_a .*= (1.0 / p_jump)`
     - `(; K0, U_residual) = _build_cptp_channel(R_a, config.delta)`
     - Store `K0s[a] = K0`, `U_residuals[a] = U_residual`
   - Return `(; K0s, U_residuals)`

   Note: Do NOT store Rs (not needed for the hot loop). The trajectory workspace stores Rs for CPTP verification tests, but the DM path only needs K0s and U_residuals.

   Add a docstring explaining: "Precompute per-jump CPTP channel matrices (K0, U_residual) for all jumps. Eliminates per-step eigendecomposition in the run_thermalize hot loop."
  </action>

   ```julia
   function _accumulate_rho_jump!(
       scratch::ThermalizeScratch{<:Complex},

       config::Config{Thermalize, EnergyDomain},

   )
   ```

   - Return nothing

   ```julia
   function _accumulate_rho_jump!(
       scratch::ThermalizeScratch{<:Complex},
       evolving_dm::Matrix{<:Complex},
       jump::JumpOp,
       hamiltonian::HamHam,
       config::Config{Thermalize, BohrDomain},

   )
   ```
   - `fill!(scratch.rho_jump, 0)`
   - `scaled_delta = config.delta * jump_weight_scaling`

     - Compute `scratch.jump_oft = alpha(hamiltonian.bohr_freqs, nu_2) * jump.in_eigenbasis`

This matches the existing `_jump_contribution!` exactly.

   ```julia
   function _apply_precomputed_channel!(
       evolving_dm::Matrix{<:Complex},
       K0::Matrix{<:Complex},
       U_residual::Matrix{<:Complex},
       scratch::ThermalizeScratch{<:Complex},
   )
   ```
   - Same body as `_finalize_kraus_step!` lines 181-191, but uses the passed-in K0 and U_residual instead of calling `_build_cptp_channel`:

     ```julia
     mul!(scratch.rho_next, scratch.sandwich_tmp, K0')

     return evolving_dm
     ```
