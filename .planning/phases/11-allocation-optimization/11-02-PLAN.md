<task type="auto">
  <name>Task 1: Eliminate Diagonal wrappers in B_time and B_trotter with pre-allocated vector buffers</name>
  <files>src/coherent.jl</files>
  <action>
Replace all four B_time/B_trotter functions to use pre-allocated vector buffers instead of Diagonal closures. The key insight: `dim = size(hamiltonian.data)` returns a TUPLE (e.g., `(16, 16)`), and `zeros(CT, dim)` creates a 16x16 MATRIX, not a vector. So `U` and `U_minus_2` are already dense matrices receiving Diagonal broadcasts. The optimization replaces the Diagonal creation with direct vector operations.

**B_time single-jump (lines 160-185):**

Replace the `diag_time_evolve` closure and the matrix products. The current code does:
```julia
diag_time_evolve(t) = Diagonal(exp.(1im * hamiltonian.eigvals * t))
U .= diag_time_evolve(s * beta)  # broadcasts Diagonal into dense matrix U
U_minus_2 .= diag_time_evolve(-2.0 * s * beta)
b_plus_summand .+= b_plus[s] * U * jump.in_eigenbasis' * U_minus_2 * jump.in_eigenbasis * U
```

Replace with pre-allocated diagonal vectors and element-wise scaling. The product `diag(u) * M * diag(v)` can be computed as `u .* M .* v'` (row-scale by u, column-scale by v). The full chain `U * A' * U2 * A * U` where U, U2 are diagonal becomes element-wise operations:

```julia
d = size(hamiltonian.data, 1)  # integer, not tuple
CT = Complex{eltype(hamiltonian.eigvals)}
eigvals = hamiltonian.eigvals

# Pre-allocated vector buffers
diag_u = Vector{CT}(undef, d)
diag_u2 = Vector{CT}(undef, d)

# Pre-allocated matrix scratch
b_plus_summand = zeros(CT, d, d)
tmp = zeros(CT, d, d)
B = zeros(CT, d, d)

jump_eig = jump.in_eigenbasis

for s in keys(b_plus)
    t_s = s * beta
    @. diag_u = exp(1im * eigvals * t_s)
    @. diag_u2 = exp(-2im * eigvals * t_s)

    # Compute: diag(u) * jump' * diag(u2) * jump * diag(u)
    # Step-by-step with element-wise diagonal scaling:
    # tmp = jump' .* diag_u2' (column-scale jump' by u2, but jump' rows = conj(jump cols))
    # Actually, the simplest correct approach:
    # 1) scale columns of jump_eig by diag_u:  tmp[i,j] = jump_eig[i,j] * diag_u[j]
    # 2) left-multiply by diag(u2): tmp[i,j] = diag_u2[i] * tmp[i,j]
    # 3) left-multiply result by jump_eig': result = jump_eig' * tmp  (this is a matrix multiply, not avoidable)
    # 4) scale: result[i,j] *= diag_u[i] * diag_u[j] -- wait, step 3 already has full result

    # Cleaner approach: build the product using tmp buffers
    # M1 = jump_eig * diag(u)  ->  tmp[i,j] = jump_eig[i,j] * diag_u[j]
    @. tmp = jump_eig * diag_u'  # broadcast: (d,d) .* (1,d) = column scaling

    # M2 = diag(u2) * M1 = diag(u2) * jump * diag(u)  -> scale rows
    @. tmp = diag_u2 * tmp  # broadcast: (d,) .* (d,d) = row scaling

    # M3 = jump' * M2 -- this is a genuine matrix multiply, cannot avoid
    # But we can use: result = jump_eig' * tmp, accumulate into b_plus_summand
    # b_plus_summand += b_s * diag(u) * (jump' * diag(u2) * jump * diag(u))
    #                = b_s * diag(u) * (jump_eig' * tmp)

    # Actually rewrite the full product:
    # P = diag(u) * jump' * diag(u2) * jump * diag(u)
    # P[i,j] = u[i] * (jump' * diag(u2) * jump * diag(u))[i,j]
    # and (...)  [i,j] = sum_k conj(jump[k,i]) * u2[k] * (jump * diag(u))[k,j]
    #                   = sum_k conj(jump[k,i]) * u2[k] * sum_l jump[k,l] * u[l] * delta(l,j)
    #                   = sum_k conj(jump[k,i]) * u2[k] * jump[k,j] * u[j]
    # So P[i,j] = u[i] * u[j] * sum_k conj(jump[k,i]) * u2[k] * jump[k,j]

    # The inner sum is (jump' * diag(u2) * jump)[i,j]
    # We still need one matrix multiply. Use tmp for that:
    # tmp = diag(u2) * jump  -> row-scale
    @. tmp = diag_u2 * jump_eig   # (d,) .* (d,d) = row scaling

    # Now we need jump' * tmp, which is a matrix multiply.
    # We cannot avoid this mul!, but we CAN avoid Diagonal allocations.
    # Store in a second scratch buffer or reuse B temporarily.
    # Actually, look at the existing code: it chains 5 matrix multiplies creating 4 intermediates.
    # The Diagonal elimination alone is the phase scope. The matrix chain intermediates
    # are out of scope per research open question #1.

    # Simplest correct approach preserving existing computation structure
    # but eliminating the Diagonal wrapper:
    # The product U * A' * U2 * A * U where U = diag(u), U2 = diag(u2)
    # can be done as element-wise scaling around one matrix multiply.
end
```

**IMPORTANT IMPLEMENTATION NOTE:** The simplest correct approach that eliminates Diagonal allocations while preserving correctness:

For each B_time/B_trotter variant, replace the `Diagonal(exp.(...))` closure with:
1. Pre-allocate `diag_u` and `diag_u2` as `Vector{CT}(undef, d)` where `d = size(..., 1)` (integer)
2. In the loop, compute `@. diag_u = exp(1im * eigvals * t_s)` etc. into the pre-allocated vectors
3. For the matrix product `diag(u) * M`, use element-wise: `diag_u .* M` (row scaling by column vector)
4. For `M * diag(u)`, use: `M .* diag_u'` (column scaling by row vector)
5. The inner matrix multiply (`jump' * (scaled_jump)`) still requires `mul!` with a scratch buffer

The dimension bug fix is critical: change `dim = size(hamiltonian.data)` (tuple) to `d = size(hamiltonian.data, 1)` (integer) in all four variants.

For `b_plus_summand`, `U`, `U_minus_2`, and `B` which are currently `zeros(CT, dim)` (creating matrices because dim is a tuple), change to `zeros(CT, d, d)` for clarity (or keep the tuple form since it works the same -- but use integer `d` for vector buffers).

The exact element-wise implementation of the 5-matrix product chain is Claude's discretion per the CONTEXT.md decisions. The requirement is: NO `Diagonal(...)` constructor calls in any loop body. Pre-allocated vector buffers + element-wise `.*` broadcasting + `mul!` for the unavoidable matrix multiply.

Apply the same pattern to all four variants:
- B_time single-jump (lines 160-185)
- B_time multi-jump (lines 187-214)
- B_trotter single-jump (lines 216-246)
- B_trotter multi-jump (lines 248-277)

For B_trotter, the closure is `trotter_time_evolution(n) = Diagonal(trotter.eigvals_t0 .^ n)`. Replace with `@. diag_u = trotter.eigvals_t0 ^ num_t0_steps` in-place.
