**DIAG-01: `extract_leading_eigendata(L::Matrix{ComplexF64}; n_modes::Int=20)`**

Implementation (per research Pattern 2):
- Dense `eigen(L)` -- full eigendecomposition (feasible for n<=6: 4096x4096)
- Sort by `sortperm(abs.(real.(F.values)))` -- proximity to zero
- Select leading `n_modes` eigenvalues and right eigenvectors
- Left eigenvectors via `inv(V_full)'` where V_full is ALL right eigenvectors in sorted order. Then select leading n_modes columns.
  - Specifically: `V_full = F.vectors[:, perm]`, `V_inv = inv(V_full)`, `V_left = conj(V_inv[1:n_modes, :]')` -- this gives left eigenvectors as columns such that `V_left' * V_right ≈ I` (biorthonormal)
  - NOTE: `inv(V)` rows are left eigenvectors. To get them as columns: transpose and conjugate. Since `L^H w = conj(lambda) w` for left eigenvectors, and `V^{-1} L = D V^{-1}`, the rows of `V^{-1}` satisfy `l_k^T L = lambda_k l_k^T`, so `l_k` (row, transposed to column) is the left eigenvector in the "transpose" sense. For the bilinear form `l_k^T r_j = delta_{kj}`, we need the unconjugated transpose. The overlap formula uses `Tr[L_k^dagger ...]` which involves the conjugate. So store `V_left_cols = V_inv[1:n_modes, :]'` (just transpose, no conjugate). Then `Tr[L_k^dagger X] = dot(conj(V_left_cols[:, k]), vec(X)) = vec(X)^H V_left_cols[:, k]`. Verify with: `V_left_cols' * V_right ≈ I_{n_modes}` (when V_left comes from rows of full inv(V)).
  - Actually simpler: `V_inv_full = inv(V_full)`. Left eigenvectors as columns = `V_inv_full'` (transpose). Select first n_modes: `V_left = V_inv_full[1:n_modes, :]'`. Then `V_left' * V_right` should be identity. For overlap formula: `Tr[L_k^dagger X] = conj(l_k)^T vec(X) = dot(V_left[:, k], vec(X))` using Julia's `dot` which conjugates the first argument.
- Compute `im_re_ratios`: for each mode k>=2, `abs(imag(lambda_k)) / max(abs(real(lambda_k)), 1e-30)`. For mode 1 (steady state), set ratio to 0.0.
- Return `EigenDecompositionResult`

**DIAG-03/04: `compute_anti_hermitian_defect(L::Matrix{ComplexF64}, gibbs::Hermitian; eps_trunc::Float64=1e-12)`**

Implementation (per research Pattern 3 -- diagonal KMS similarity):
- Extract Gibbs diagonal: `gibbs_diag = real.(diag(Matrix(gibbs)))` -- Gibbs is diagonal in eigenbasis
- Truncate: `gibbs_diag_safe = max.(gibbs_diag, eps_trunc)`
- `rho_quarter = gibbs_diag_safe .^ 0.25`; `rho_inv_quarter = gibbs_diag_safe .^ (-0.25)`
- Diagonal KMS transform: `d_right = kron(rho_quarter, rho_quarter)`, `d_left = kron(rho_inv_quarter, rho_inv_quarter)`
- `D_matrix = d_left .* L .* d_right'` (element-wise broadcasting: d_left as column vector, d_right' as row vector)
- Hermitian/anti-Hermitian split: `H_part = (D_matrix + D_matrix') / 2`, `A_part = (D_matrix - D_matrix') / 2`
- `A_norm = opnorm(A_part)` (operator 2-norm = largest singular value)
- `H_eigenvalues = eigvals(Hermitian(H_part))` -- sorted ascending by default
- `H_gap = abs(H_eigenvalues[2])` if sorted by absolute value, but `eigvals(Hermitian(...))` returns sorted ascending. Need second-smallest absolute value: `sorted_H = sort(abs.(H_eigenvalues))`, `H_gap = sorted_H[2]`.
  - Actually: the Hermitian part has a zero eigenvalue (steady state). The gap is the second smallest absolute eigenvalue.
- `defect_ratio = A_norm / max(H_gap, 1e-30)` -- guard against division by zero
- Warning threshold = 0.1 (per discretion recommendation from research)
- `warning = defect_ratio > 0.1`
- If warning: `@warn "Anti-Hermitian defect ratio $(defect_ratio) > 0.1 threshold. Non-normality effects may cause oscillatory transients -- consider oscillatory fit model."`
- Return `DefectResult`

**DIAG-06: `compute_sz_labels(eigen_result, hamiltonian; n_modes=20)`**

Implementation (per research Pattern 4):
- Compute total Sz operator in Hamiltonian eigenbasis:
  - `n = Int(log2(dim))` where `dim = size(hamiltonian.data, 1)`
  - Build `Sz_comp = sum(pad_term([Z], n, site) for site in 1:n) / 2`
  - Transform to eigenbasis: `Sz_eigen = V' * Sz_comp * V` where `V = hamiltonian.eigvecs`
  - `sz_vals = real.(diag(Sz_eigen))` -- Sz eigenvalue for each energy eigenstate
- For each mode k (1 to n_modes):
  - `M_k = reshape(eigen_result.right_eigenvectors[:, k], dim, dim)`
  - `weights = abs2.(M_k)` -- element-wise |M_k[i,j]|^2
  - Build Delta_Sz weight map: for each (i,j) with weight > 1e-14, compute `dsz = round(sz_vals[i] - sz_vals[j]; digits=6)` and accumulate weight
  - Find dominant sector: maximum weight entry
  - `purity = dominant_weight / total_weight`
  - `is_pure = purity > 0.95` (per discretion recommendation)
  - Return `SzSectorLabel(dominant_dsz, purity, is_pure, sector_weights_dict)`
