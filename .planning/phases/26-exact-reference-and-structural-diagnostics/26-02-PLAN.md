**Modify `eigenbasis_overlap_analysis` in `src/gap_estimation.jl`:**

The current implementation uses a simplified formula:
```julia
alpha = V \ reshape(rho0, :)  # implicit left eigenvectors
coeffs[i, k] = dot(O_vec, V[:, k]) * alpha[k]  # Tr[O^H R_k] * (V^{-1} rho0)_k
```

This is close but differs from the supplementary info formula in two ways:
1. It uses `rho0` instead of `(rho_0 - rho_beta)` -- the steady-state subtraction is implicit via lambda_1=0
2. It doesn't explicitly compute left eigenvectors

Update to the correct formula per locked decision:
`c_k = Tr[O R_k] * Tr[L_k^dagger (rho_0 - rho_beta)]`

**New implementation:**

Add a `rho_beta` keyword argument (optional, with default `nothing` for backward compatibility):

```julia
function eigenbasis_overlap_analysis(
    L::Matrix{<:Complex},
    observables::Vector{<:Matrix{<:Complex}},
    observable_names::Vector{String},
    rho0::Matrix{<:Complex};
    rho_beta::Union{Hermitian, Nothing}=nothing,  # NEW: Gibbs state for proper subtraction
)
```

If `rho_beta` is provided, use the correct formula:
1. `F = eigen(L)`, sort by |Re|
2. `V_right = F.vectors[:, perm]` (all right eigenvectors, sorted)
3. `V_inv = inv(V_right)` -- rows are left eigenvectors
4. `V_left = V_inv'` -- left eigenvectors as columns (transpose, not conjugate transpose)
5. `rho_diff = rho0 - Matrix(rho_beta)` -- explicit subtraction
6. For each mode k, observable i, compute using the correct formula `c_k = Tr[O_i R_k] * Tr[L_k^dagger (rho_0 - rho_beta)]`:
   ```julia
   dim = isqrt(size(V_right, 1))
   R_k_mat = reshape(V_right[:, k], dim, dim)  # Right eigenvector as dim x dim operator
   L_k_vec = V_left[:, k]                       # Left eigenvector (vectorized)
   # Factor 1: Tr[O_i R_k] = tr(observables[i] * R_k_mat)
   # Factor 2: Tr[L_k^dagger (rho_0 - rho_beta)] = dot(L_k_vec, vec(rho_diff))
   #   (Julia's dot conjugates first arg: dot(a,b) = conj(a)'*b = a^H b)
   coeffs[i, k] = tr(observables[i] * R_k_mat) * dot(L_k_vec, vec(rho_diff))
   ```

If `rho_beta` is `nothing` (backward compatibility): use the original formula `alpha = V \ vec(rho0)`, `coeffs[i, k] = dot(O_vec, V[:, k]) * alpha[k]`. This preserves existing behavior for callers that don't pass rho_beta.

**Update the docstring** to document:
- New optional `rho_beta` keyword for proper steady-state subtraction
- When `rho_beta` is provided, uses the exact formula with left+right eigenvectors
- When `rho_beta` is omitted, uses the original simplified formula (v1.3 behavior)

**Update OverlapAnalysisResult** computation accordingly: the eigenvalues, exact_gap, gap_overlap, and rel_overlap computations remain the same structurally, just use the updated coefficients.
