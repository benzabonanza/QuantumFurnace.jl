    using StableRNGs

    @testset "Mixing Time Estimation" begin

    **MIX-01: Exponential fit on synthetic trace distance curve**
    - Create synthetic data: `A_true=1.5, gap_true=0.3, C_true=0.001`, `times=0.0:0.1:50.0`, `dists = A .* exp.(-gap .* times) .+ C`
    - Build a ThermalizeResults manually: use `make_config(Thermalize(), EnergyDomain(); mixing_time=50.0)` from test_helpers.jl, create a dummy `final_dm` as zeros(ComplexF64, DIM, DIM) where DIM comes from the config's num_qubits (2^num_qubits), and pass dists/times/metadata.
    - NOTE on ThermalizeResults construction: Check the struct definition -- it requires `config::Config`, `final_dm::Matrix{Complex{T}}`, `trace_distances::Vector{T}`, `time_steps::Vector{T}`, `metadata::Dict{Symbol,Any}`.     - Call `estimate_mixing_time(result; skip_initial=0.1)`
    - Assert: `isapprox(est.fitted_gap, gap_true; atol=0.01)`, `est.r_squared > 0.99`, `est.converged == true`
    - Assert: `est.amplitude > 0`, `est.offset >= 0`, `est.fit_result isa FitResult`

    **MIX-02: Extrapolation mode**
    - Same synthetic data, call with `extrapolate=true, target_epsilon=0.01`
    - Compute expected: `t_expected = -log((0.01 - C_true) / A_true) / gap_true`
    - Assert: `isapprox(est.mixing_time_extrapolated, t_expected; rtol=0.05)`
    - Assert: `est.mixing_time == est.mixing_time_extrapolated` (primary answer is extrapolated when extrapolate=true)
    - Assert: `est.target_epsilon == 0.01`

    **MIX-03: Actual mixing time from data (no extrapolation)**
    - Synthetic data long enough that trace distance crosses 0.01
    - Call with `target_epsilon=0.01, extrapolate=false`
    - Assert: `est.mixing_time_actual !== nothing`
    - Assert: `est.mixing_time_actual` is approximately the time where the synthetic curve first crosses 0.01
    - Assert: `est.mixing_time == est.mixing_time_actual` (primary answer is actual when extrapolate=false)
    - Assert: `est.mixing_time_extrapolated === nothing`

    **MIX-04: skip_initial keyword**
    - Create data with fast transient added: `dists = 2.0 .* exp.(-0.3 .* times) .+ 0.5 .+ 3.0 .* exp.(-2.0 .* times)`
    - Compare `skip_initial=0.0` vs `skip_initial=0.3`
    - Assert: skip_initial=0.3 gives a fitted_gap closer to 0.3 than skip_initial=0.0

    **MIX-05: MixingTimeEstimate struct fields**
    - Assert all expected fieldnames exist: `fitted_gap, amplitude, offset, gap_ci, gap_se, r_squared, converged, mixing_time, mixing_time_extrapolated, mixing_time_actual, target_epsilon, fit_result`
    - Assert types: `est.gap_ci isa Tuple{Float64, Float64}`, `est.fit_result isa FitResult`, etc.

    **MIX-07: Quality gate warnings**
    - Test R^2 warning: create bad synthetic data (e.g., `sin.(times)` as trace distances, patched to be positive). Use `Test.@test_warn` to verify the R^2 < 0.95 warning fires.
    - Test offset warning: create data with large C (e.g., C=1.0) and call with target_epsilon=0.5. Use `Test.@test_warn` to verify the offset warning fires.
    - Test non-convergence warning: this is hard to trigger reliably; skip if needed or use a very short/pathological dataset.

    **Edge cases:**
    - `extrapolate=true` without `target_epsilon`: should throw ArgumentError
    - Fewer than 10 data points: should throw ArgumentError
    - Target not reached in data: `mixing_time_actual` should be `nothing`

    **Integration test: real run_thermalize output**
    - Use `make_config(Thermalize(), EnergyDomain(); mixing_time=5.0)` with N3 test system

    - This is the key integration test proving the API works end-to-end
