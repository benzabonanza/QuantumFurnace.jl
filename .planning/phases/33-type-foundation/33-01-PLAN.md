  <action>
In src/structs.jl, make the following changes:

1. **Remove the old abstract types and config structs.** Delete:
   - `abstract type AbstractConfig{D<:AbstractDomain, T<:AbstractFloat} end`
   - `abstract type AbstractLiouvConfig{D<:AbstractDomain, T<:AbstractFloat} <: AbstractConfig{D,T} end`
   - `abstract type AbstractThermalizeConfig{D<:AbstractDomain, T<:AbstractFloat} <: AbstractConfig{D,T} end`
   - The entire `@kwdef struct LiouvConfig{D,T}` definition
   - The entire `@kwdef struct LiouvConfigGNS{D,T}` definition and its outer constructor
   - The entire `@kwdef struct ThermalizeConfig{D,T}` definition
   - The entire `@kwdef struct ThermalizeConfigGNS{D,T}` definition and its outer constructor

2. **Add the simulation singleton hierarchy** (after the domain definitions, before LindbladianWorkspace):
   ```julia
   # Simulation types
   abstract type AbstractSimulation end
   struct Lindbladian    <: AbstractSimulation end
   struct Thermalize     <: AbstractSimulation end
   struct KrylovSpectrum <: AbstractSimulation end
   struct Trajectory     <: AbstractSimulation end
   ```

3. **Add the construction singleton hierarchy** (right after simulation types):
   ```julia
   # Construction types (detailed balance)
   abstract type AbstractConstruction end
   struct KMS <: AbstractConstruction end
   struct GNS <: AbstractConstruction end
   struct DLL <: AbstractConstruction end
   ```

4. **Add the with_coherent trait function** (right after construction types):
   ```julia
   # Trait: coherent term presence (derived from construction type)
   with_coherent(::KMS) = true
   with_coherent(::GNS) = false
   with_coherent(::DLL) = true  # placeholder for Ding et al.
   ```

5. **Define the unified Config struct** (replaces all 4 old config types). Use `@kwdef` with this exact field ordering:
   ```julia
   @kwdef struct Config{S <: AbstractSimulation, D <: AbstractDomain, C <: AbstractConstruction, T <: AbstractFloat}
       # Type-encoding singletons
       sim::S
       domain::D
       construction::C

       # System parameters
       num_qubits::Int
       with_linear_combination::Bool

       # Physics parameters
       beta::T
       sigma::T
       gaussian_parameters::Union{Tuple{T, T}, Tuple{Nothing, Nothing}} = (nothing, nothing)
       a::Union{T, Nothing} = nothing
       b::Union{T, Nothing} = nothing

       # Grid parameters
       num_energy_bits::Union{Int, Nothing} = nothing
       t0::Union{T, Nothing} = nothing
       w0::Union{T, Nothing} = nothing
       eta::Union{T, Nothing} = nothing
       num_trotter_steps_per_t0::Union{Int, Nothing} = nothing

       # Thermalize-specific
       mixing_time::Union{T, Nothing} = nothing
       delta::Union{T, Nothing} = nothing
   end
   ```

   **Important:** The "no defaults" decision means no defaults for required params (sim, domain, construction, num_qubits, beta, sigma, with_linear_combination). The `nothing` defaults for optional fields (a, b, grid params, mixing_time, delta) are kept because they are functional necessities for domains/simulations that don't use those fields.

6. **Add an explicit outer constructor** to ensure type inference works correctly with 4 type parameters. Place this right after the struct definition:
   ```julia
   function Config(;
       sim::S, domain::D, construction::C,
       beta::T,
       kwargs...
   ) where {S <: AbstractSimulation, D <: AbstractDomain, C <: AbstractConstruction, T <: AbstractFloat}
       Config{S, D, C, T}(; sim=sim, domain=domain, construction=construction, beta=beta, kwargs...)
   end
   ```

Keep everything else in structs.jl (LindbladianWorkspace, JumpOp, DMSimulationResult, LindbladianResult, ConvergenceData, LSIFramework, OFTCaches) unchanged. The docstring from LiouvConfig can be adapted as a docstring for Config (update field descriptions to remove with_coherent field, add sim/construction fields).
