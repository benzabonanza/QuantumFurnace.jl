**1. `_accumulate_dissipator_2op!(out, A, B_dag, rho, scalar, ws)`**

Two-operator dissipator for BohrDomain forward:
```
scalar * D(A, B_dag)(rho) = scalar * (A * rho * B_dag' - 0.5*(B_dag * A * rho + rho * B_dag * A))
```

Where:
- `A` = alpha_A_nu1 (dense Matrix{T}, the alpha-weighted eigenbasis)
- `B_dag` = A_nu2_dag (could be dense Matrix{T} -- built via scatter into ws.tmp2)
- The "right" operator in the sandwich is `B_dag'` (adjoint of B_dag)
- The anticommutator product is `B_dag * A`

Implementation using BLAS.gemm!:
```julia
function _accumulate_dissipator_2op!(
    out::Matrix{T}, A::Matrix{T}, B_dag::Matrix{T}, rho::Matrix{T}, scalar::Real, ws
) where {T<:Complex}
    CT = one(T); ZT = zero(T)
    # B_dag * A -> ws.LdagL (anticommutator product)
    BLAS.gemm!('N', 'N', CT, B_dag, A, ZT, ws.LdagL)
    # Term 1: scalar * A * rho * B_dag'
    BLAS.gemm!('N', 'N', CT, A, rho, ZT, ws.tmp1)        # tmp1 = A * rho
    BLAS.gemm!('N', 'C', CT, ws.tmp1, B_dag, ZT, ws.tmp2) # tmp2 = A * rho * B_dag'
    BLAS.axpy!(T(scalar), ws.tmp2, out)
    # Term 2: -0.5 * scalar * B_dag*A * rho
    BLAS.gemm!('N', 'N', CT, ws.LdagL, rho, ZT, ws.tmp1)
    BLAS.axpy!(T(-0.5 * scalar), ws.tmp1, out)
    # Term 3: -0.5 * scalar * rho * B_dag*A
    BLAS.gemm!('N', 'N', CT, rho, ws.LdagL, ZT, ws.tmp1)
    BLAS.axpy!(T(-0.5 * scalar), ws.tmp1, out)
end
```

NOTE on scratch usage: This function uses ws.tmp1, ws.tmp2, and ws.LdagL. The caller must ensure A_nu2_dag is NOT in any of these scratch buffers when this function is called. Per the research recommendations, A_nu2_dag should be built into a SEPARATE buffer. Since we need tmp2 inside the helper, the caller must use a different strategy.

**CRITICAL: The scratch conflict.** The 2op helper uses ws.tmp2 for the sandwich term `A * rho * B_dag'`. So we CANNOT build A_nu2_dag into ws.tmp2 as the research initially recommended. Instead:
- Build A_nu2_dag into a local dense matrix (allocated once outside the loop, zeroed each iteration). Since BohrDomain allows some allocations (per CONTEXT.md), allocate ONE dim x dim Matrix{T} at the start of `apply_lindbladian!` for the BohrDomain method and reuse it across iterations by zeroing + scatter. This is ONE allocation per matvec call (acceptable for Bohr).
- OR, restructure the 2op helper to avoid using ws.tmp2 for the sandwich, computing it differently. Since the helper needs 3 temporaries (anticommutator product, 2 intermediate muls) and we have 3 scratch matrices (tmp1, tmp2, LdagL), there's no room for A_nu2_dag in scratch.

**Recommended approach (Claude's discretion):** Allocate one `A_nu2_dag = zeros(CT, dim, dim)` at the top of the BohrDomain `apply_lindbladian!` method (outside the loop). Zero and scatter each iteration. This is ONE allocation per matvec call, which is well within the "unavoidable allocations" budget for BohrDomain.

**2. `_accumulate_adjoint_dissipator_2op!(out, A, B_dag, rho, scalar, ws)`**

CRITICAL: The CONTEXT.md suggests "swap arguments at call site" for the adjoint, BUT the research proves this is mathematically INCORRECT (see RESEARCH.md Pitfall 5 and Open Question 1). The researcher notes "researcher should verify the math" -- and the research HAS verified it.

The Hilbert-Schmidt adjoint of D(A, B_dag) is:
```
D*(A, B_dag)(rho) = A' * rho * B_dag - 0.5*(A' * B_dag * rho + rho * A' * B_dag)
```

Note the differences from forward:
- Sandwich: `A * rho * B_dag'` becomes `A' * rho * B_dag`
- Anticommutator: `B_dag * A` becomes `A' * B_dag`

Simple swap of A and B_dag would give `B_dag * rho * A'` in the sandwich (WRONG: should be `A' * rho * B_dag`).

Implementation:
```julia
function _accumulate_adjoint_dissipator_2op!(
    out::Matrix{T}, A::Matrix{T}, B_dag::Matrix{T}, rho::Matrix{T}, scalar::Real, ws
) where {T<:Complex}
    CT = one(T); ZT = zero(T)
    # A' * B_dag -> ws.LdagL (adjoint anticommutator product)
    BLAS.gemm!('C', 'N', CT, A, B_dag, ZT, ws.LdagL)
    # Term 1: scalar * A' * rho * B_dag
    BLAS.gemm!('C', 'N', CT, A, rho, ZT, ws.tmp1)          # tmp1 = A' * rho
    BLAS.gemm!('N', 'N', CT, ws.tmp1, B_dag, ZT, ws.tmp2)   # tmp2 = A' * rho * B_dag
    BLAS.axpy!(T(scalar), ws.tmp2, out)
    # Term 2: -0.5 * scalar * A'*B_dag * rho
    BLAS.gemm!('N', 'N', CT, ws.LdagL, rho, ZT, ws.tmp1)
    BLAS.axpy!(T(-0.5 * scalar), ws.tmp1, out)
    # Term 3: -0.5 * scalar * rho * A'*B_dag
    BLAS.gemm!('N', 'N', CT, rho, ws.LdagL, ZT, ws.tmp1)
    BLAS.axpy!(T(-0.5 * scalar), ws.tmp1, out)
end
```

HOWEVER: First try the simple swap approach (swap A and B_dag at the call site). If the round-trip test against `L_dense'` passes, the simple swap is correct for this specific BohrDomain operator structure, and the separate helper is unnecessary. If it fails, implement the separate adjoint helper above. The round-trip test is the ultimate arbiter. **3. `apply_lindbladian!(ws, rho, config::AbstractLiouvConfig{BohrDomain}, hamiltonian)`**

BohrDomain forward matvec -- fundamentally different loop structure from Energy/Time/Trotter:

```julia
function apply_lindbladian!(
    ws::KrylovWorkspace{T}, rho::Matrix{T},
    config::AbstractLiouvConfig{BohrDomain}, hamiltonian::HamHam
) where {T<:Complex}
    (; alpha, gamma_norm_factor) = ws.precomputed_data
    dim = size(rho, 1)
    fill!(ws.rho_out, 0)

    # Coherent term (identical to other domains)
    B = ws.B_total
    if B !== nothing
        mul!(ws.tmp1, B, rho)
        @. ws.rho_out += -1im * ws.tmp1
        mul!(ws.tmp1, rho, B)
        @. ws.rho_out += 1im * ws.tmp1
    end

    # Allocate A_nu2_dag buffer (one allocation per matvec -- acceptable for Bohr)
    A_nu2_dag = zeros(T, dim, dim)

    for (k, eigenbasis) in enumerate(ws.jump_eigenbases)
        for nu_2 in keys(hamiltonian.bohr_dict)
            # alpha_A: dense alpha-weighted eigenbasis
            @. ws.jump_oft = alpha(hamiltonian.bohr_freqs, nu_2) * eigenbasis

            # Build A_nu2_dag via index scatter (avoid sparse() allocation)
            fill!(A_nu2_dag, 0)
            indices = hamiltonian.bohr_dict[nu_2]
            @inbounds for idx in indices
                i = idx[1]; j = idx[2]
                A_nu2_dag[j, i] = conj(eigenbasis[i, j])
            end

            _accumulate_dissipator_2op!(ws.rho_out, ws.jump_oft, A_nu2_dag, rho, gamma_norm_factor, ws)
        end
    end

    return ws.rho_out
end
```

Key notes:
- BohrDomain has NO Hermitian half-grid optimization (no energy_labels loop)
- BohrDomain has NO transition function (alpha encodes the frequency dependence)
- BohrDomain uses `eigenbasis` (from `ws.jump_eigenbases[k]`) not `jump.in_eigenbasis` (avoids abstract boxing)
- A_nu2_dag is built by scattering `conj(eigenbasis[i,j])` to position `[j,i]` (the dagger operation: swap indices and conjugate)
- gamma_norm_factor is the scalar (no additional prefactor formula)
