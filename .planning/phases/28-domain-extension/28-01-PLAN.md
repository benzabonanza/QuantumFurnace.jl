<task type="auto">
  <name>Task 1: Implement Time/Trotter forward and adjoint matvec methods</name>
  <files>src/krylov_matvec.jl</files>
  <action>
Add two new dispatch methods at the end of krylov_matvec.jl (before the closing of the file):

**1. `apply_lindbladian!(ws, rho, config::AbstractLiouvConfig{D}, hamiltonian) where {D<:Union{TimeDomain, TrotterDomain}}`**

Mirror the EnergyDomain `apply_lindbladian!` line-by-line with these differences:
- Destructure `ws.precomputed_data` to include `oft_nufft_prefactors` (in addition to `transition`, `gamma_norm_factor`, `energy_labels`)
- Replace `_krylov_oft!(ws.jump_oft, eigenbasis, bohr_freqs, w, inv_4sigma2)` with:
  ```julia
  nufft_prefactor_matrix = _prefactor_view(oft_nufft_prefactors, w)
  @. ws.jump_oft = eigenbasis * nufft_prefactor_matrix
  ```
- Replace the scalar prefactor formula. Instead of `(config.w0 / (config.sigma * sqrt(2 * pi))) * gamma_norm_factor`, use:
  ```julia
  prefactor = config.w0 * config.t0^2 * (config.sigma * sqrt(2 / pi)) / (2 * pi) * gamma_norm_factor
  ```
  This comes directly from `jump_workers.jl:109`.
- Keep the coherent term block (`-i[B, rho]`) IDENTICAL to EnergyDomain -- it uses `ws.B_total` and `ws.tmp1` which are domain-agnostic
- Keep the Hermitian half-grid optimization and dissipator helper calls IDENTICAL
- Do NOT access `bohr_freqs` or `inv_4sigma2` (these are only needed for `_krylov_oft!` in EnergyDomain)

Use the same BLAS primitives as EnergyDomain (BLAS.gemm!/axpy!, concrete-typed `ws.jump_eigenbases`). The existing dissipator helpers (`_accumulate_dissipator!`, `_accumulate_dissipator_adj_L!`) are reused unchanged.

**2. `apply_adjoint_lindbladian!(ws, rho, config::AbstractLiouvConfig{D}, hamiltonian) where {D<:Union{TimeDomain, TrotterDomain}}`**

Same as forward but with EnergyDomain adjoint changes:
- Coherent term: `+i[B, rho]` (sign flip)
- Dissipator: use `_accumulate_adjoint_dissipator!` and `_accumulate_adjoint_dissipator_adj_L!`
- Same NUFFT prefactor computation and same scalar prefactor as forward (no prefactor changes for adjoint, per CONTEXT.md)

Add appropriate docstrings referencing the dense `_jump_contribution!` for `Union{TimeDomain, TrotterDomain}` and noting the NUFFT prefactor difference.

IMPORTANT anti-patterns to avoid:
- Do NOT use `mul!` with Adjoint wrappers (causes 16-byte boxing allocation)
- Do NOT access `jump.in_eigenbasis` (abstract type causes 832-byte boxing allocation)
- Do NOT call `_krylov_oft!` (that's EnergyDomain-only)
  </action>
  <verify>
Run `julia -e 'using Pkg; Pkg.test()'` to confirm the test suite still passes (existing tests should not break).
Also verify the new methods parse: `julia -e 'include("src/krylov_matvec.jl")' 2>&1` should show no errors (approximate check -- actual verification is in Task 2).
  </verify>
  <done>
Two new method signatures exist in krylov_matvec.jl dispatching on `Union{TimeDomain, TrotterDomain}` -- one for forward, one for adjoint. Both use `_prefactor_view` for NUFFT OFT and the `t0^2` prefactor formula.
  </done>
