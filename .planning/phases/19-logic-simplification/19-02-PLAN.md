<objective>
Flatten the trajectory call chain from 5 levels to 3 levels.

Purpose: The current call chain is `run_experiment() -> run_trajectories_adaptive() -> run_trajectories() -> _evolve_along_trajectory!() -> step_along_trajectory!()` -- 5 levels deep. The adaptive runner calls `run_trajectories()` per batch, which rebuilds the `TrajectoryFramework` every time (identical, immutable data). This wastes computation. Additionally, `_evolve_along_trajectory!` is a trivial 13-line wrapper that adds no abstraction value.

Output: A 3-level architecture: (1) public API (run_trajectories / run_trajectories_adaptive), (2) batch execution with pre-built framework, (3) step_along_trajectory! (unchanged hot path).
</objective>

**Step 1: Inline `_evolve_along_trajectory!` into `_run_chunk_no_obs!` and `_run_chunk_with_obs!`.**

Currently `_run_chunk_no_obs!` calls `_evolve_along_trajectory!(psi, fw, ws, rng, total_time)` which just does:
```julia
num_steps = ceil(Int, total_time / delta)
normalize(psi)
for _ in 1:num_steps
    step_along_trajectory!(psi, fw, ws, rng)
end
```

Inline this directly into `_run_chunk_no_obs!`. The `total_time` parameter must be added to the chunk runners (it's currently passed through to `_evolve_along_trajectory!`). The resulting code in `_run_chunk_no_obs!` becomes:

```julia
function _run_chunk_no_obs!(
    ws::TrajectoryWorkspace{<:Complex},
    fw::TrajectoryFramework{<:Complex},
    psi0::Vector{<:Complex},
    chunk::UnitRange{Int},
    master_seed::Int,
    total_time::Real,
)
    delta = fw.delta
    num_steps = ceil(Int, total_time / delta)
    psi = copy(psi0)
    for traj_id in chunk
        rng = Random.Xoshiro(master_seed + traj_id)
        copyto!(psi, psi0)
        # Normalize once per trajectory
        psi_norm2 = _norm2(psi)
        rmul!(psi, 1.0 / sqrt(max(psi_norm2, eps(Float64))))
        # Step loop (was _evolve_along_trajectory!)
        @inbounds for _ in 1:num_steps
            step_along_trajectory!(psi, fw, ws, rng)
        end
        _accumulate_density_matrix!(ws.rho_acc, psi)
    end
    return nothing
end
```

Do the same for `_run_chunk_with_obs!` -- it already has its own step loop, so just add the normalization that `_evolve_along_trajectory!` was doing and remove the call to `_evolve_along_trajectory!` if present. Looking at the code, `_run_chunk_with_obs!` already inlines the step loop and normalizes, so it needs no change.

After inlining, **delete `_evolve_along_trajectory!`** entirely.
