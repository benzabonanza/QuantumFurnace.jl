<objective>
Simplify the result struct hierarchy with clean, per-method result types.

Purpose: The current result hierarchy has several pain points: (1) TrajectoryResult has Union{Nothing,...} fields for times/measurements, (2) convergence/adaptive runners return a tuple (TrajectoryResult, ConvergenceData) instead of a single coherent value, (3) HotAlgorithmResults and HotSpectralResults carry full hamiltonian and config objects (heavy for no benefit), and (4) the names are unclear.

Output: Three clean result types: LindbladianResult (replaces HotSpectralResults), DMSimulationResult (replaces HotAlgorithmResults), and a simplified TrajectoryResult with embedded ConvergenceData. Serialization updated for backward compatibility.
</objective>

**Part A: Rename and slim HotSpectralResults -> LindbladianResult**

In `src/structs.jl`, replace:
```julia
@kwdef struct HotSpectralResults{D, T<:AbstractFloat}
    data::Matrix{Complex{T}}
    fixed_point::Matrix{Complex{T}}
    gap_mode::Matrix{Complex{T}}
    spectral_gap::Complex{T}
    hamiltonian::HamHam{T}
    trotter::Union{TrottTrott{T}, Nothing} = nothing
    config::AbstractLiouvConfig{D,T}
end
```
with:
```julia
@kwdef struct LindbladianResult{T<:AbstractFloat}
    liouvillian::Matrix{Complex{T}}
    fixed_point::Matrix{Complex{T}}
    gap_mode::Matrix{Complex{T}}
    spectral_gap::Complex{T}
end
```

Keep the `@kwdef` pattern. Drop `hamiltonian`, `trotter`, and `config` fields -- callers already have these at the call site. Rename `data` to `liouvillian` for clarity. Remove the `D` type parameter (no longer needed without config).

In `src/furnace.jl` `run_lindbladian()`, update the result construction:
```julia
result = LindbladianResult(
```

**Part F: Update serialization in results.jl**

Update `_trajectory_to_dict` to serialize the new `convergence` field:
```julia
function _trajectory_to_dict(traj::TrajectoryResult)
    d = Dict{Symbol, Any}(
        :rho_mean          => Matrix(traj.rho_mean),
        :n_trajectories    => traj.n_trajectories,
        :seed              => traj.seed,
        :times             => traj.times,
        :measurements_mean => traj.measurements_mean,
    )
    if traj.convergence !== nothing
        d[:convergence] = _convergence_to_dict(traj.convergence)
    end
    return d
end
```

Update `_dict_to_experiment` to reconstruct the convergence field:
```julia
conv = if haskey(traj_d, :convergence) && traj_d[:convergence] !== nothing
    _dict_to_convergence(traj_d[:convergence])
else
    nothing
end
traj = TrajectoryResult(
    traj_d[:rho_mean],
    traj_d[:n_trajectories],
    traj_d[:seed],
    get(traj_d, :times, nothing),
    get(traj_d, :measurements_mean, nothing),
    conv,
)
```

This is backward-compatible: old BSON files without `:convergence` key load with `convergence=nothing`.

Also check if `ExperimentResult` or `run_sweep.jl` needs to handle the ConvergenceData differently now that it's embedded. The sweep script currently stores convergence data separately in metadata via `_convergence_to_dict(conv_data)`. With the new structure, `conv_data` is inside `traj_result.convergence`, so the sweep script access pattern changes from:
```julia

```
to:
```julia

conv_data = traj_result.convergence
```
