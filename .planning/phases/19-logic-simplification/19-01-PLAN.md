<objective>
Eliminate redundant jump basis transforms by fixing JumpOp construction at the source.

Purpose: Currently, JumpOps are always built with `hamiltonian.eigvecs` as the basis, then re-transformed via `transform_jumps_to_basis(jumps, trotter.eigvecs)` at 6 call sites across 3 files for TrotterDomain. This is wasteful and error-prone. By constructing JumpOps with the correct basis from the start, all downstream transforms become unnecessary.

Output: All JumpOp construction sites use domain-aware basis selection. All `transform_jumps_to_basis` calls in the TrotterDomain internal path are removed.
</objective>

This task has two halves: (A) update JumpOp construction, (B) remove downstream transforms.

**IMPORTANT CONSTRAINT:** The test helpers `make_test_system()` and `make_small_test_system()` do NOT take a domain parameter -- they build JumpOps for general use across ALL domain tests. Since the same JumpOps are used for EnergyDomain, TimeDomain, AND TrotterDomain tests, they CANNOT be made domain-specific without breaking the test architecture.

Therefore, the fix strategy is:
- Test helpers and sweep script: Keep constructing JumpOps with `hamiltonian.eigvecs` (this is correct for Energy/Time/Bohr domains and is the "default" basis).
- `build_trotter_system()` in run_sweep.jl: This function is ONLY used for TrotterDomain. Update it to accept an optional `trotter` parameter and use `trotter.eigvecs` for the basis transform when provided. Actually, looking at the code, `build_trotter_system` creates the TrottTrott first, then the jumps -- so it CAN use `trotter.eigvecs` directly. Update it:
  ```julia
  in_eigen = trotter.eigvecs' * op * trotter.eigvecs
  ```
  instead of the current `hamiltonian.eigvecs' * op * hamiltonian.eigvecs`.

Wait -- this is WRONG. The test helpers create JumpOps used by both TrotterDomain and non-TrotterDomain tests. The sweep script's `build_trotter_system` creates JumpOps only for TrotterDomain. But the convergence/adaptive runners in convergence.jl pass these same jumps through `run_trajectories`, which calls `build_trajectoryframework`, which currently does the transform.

**Revised strategy (correct approach):**

Since test JumpOps must serve all domains, we CANNOT change test helpers. The fix applies to:

1. **`experiments/run_sweep.jl` -- `build_trotter_system()`**: This builds JumpOps exclusively for TrotterDomain. Change `in_eigen = hamiltonian.eigvecs' * op * hamiltonian.eigvecs` to `in_eigen = trotter.eigvecs' * op * trotter.eigvecs`. This means the JumpOps from `build_trotter_system()` arrive at `build_trajectoryframework()` already in Trotter basis.

2. **`src/trajectories.jl` -- `build_trajectoryframework()`** (lines 92-98): The current code does:
   ```julia
   jumps_for_diss_raw = if config.domain isa TrotterDomain && ham_or_trott isa TrottTrott
       transform_jumps_to_basis(jumps, ham_or_trott.eigvecs)
   else
       collect(JumpOp, jumps)
   end
   jumps_for_diss = convert(Vector{JumpOp{Matrix{CT}}}, jumps_for_diss_raw)
   ```
   Change to simply:
   ```julia
   jumps_for_diss = convert(Vector{JumpOp{Matrix{CT}}}, collect(JumpOp, jumps))
   ```
   No conditional transform needed -- jumps arrive in the correct basis.

3. **`src/furnace.jl` -- `construct_lindbladian()`** (lines 80-84): Remove the TrotterDomain transform block:
   ```julia
   jumps_for_diss = if config.domain isa TrotterDomain
       transform_jumps_to_basis(jumps, trotter.eigvecs)
   else
       jumps
   end
   ```
   Replace with: `jumps_for_diss = jumps`

4. **`src/furnace.jl` -- `run_thermalization()`** (lines 127-131): Same pattern -- remove the conditional:
   ```julia
   jumps_for_diss = if config.domain isa TrotterDomain
       transform_jumps_to_basis(jumps, trotter.eigvecs)
   else
       jumps
   end
   ```
   Replace with: `jumps_for_diss = jumps`

5. **`src/coherent.jl` -- `_precompute_coherent_total_B()`** (line 30): The TrotterDomain branch currently does:
   ```julia
   trotter_jumps = transform_jumps_to_basis(jumps, ham_or_trott.eigvecs)
   B = B_trotter(trotter_jumps, ham_or_trott, b_minus, b_plus, config.beta, config.sigma)
   ```
   Change to pass `jumps` directly (they're already in Trotter basis):
   ```julia
   B = B_trotter(jumps, ham_or_trott, b_minus, b_plus, config.beta, config.sigma)
   ```

6. **`src/coherent.jl` -- `_precompute_coherent_unitary_terms()`** (lines 82-86): Same -- remove:
   ```julia
   trotter_jumps = transform_jumps_to_basis(jumps, trotter.eigvecs)
   ```
   and iterate over `jumps` directly instead of `trotter_jumps`.

7. **`src/coherent.jl` -- `_precompute_coherent_terms()`** (lines 138-142): Same pattern -- remove the `trotter_jumps` variable and use `jumps` directly.

**CRITICAL: The existing tests use JumpOps built with hamiltonian.eigvecs.** Those tests exercise Energy/Time/Bohr domains where hamiltonian.eigvecs IS the correct basis. For TrotterDomain tests, the test code must still work. Check how TrotterDomain tests create their JumpOps:
