<task type="auto">
  <name>Task 1: Write the complete benchmark script</name>
  <files>simulations/main_krylov_benchmark.jl</files>
  <action>
Create `simulations/main_krylov_benchmark.jl` -- a standalone Julia script that benchmarks `krylov_spectral_gap()` at multiple system sizes and generates a scaling report.

**Script structure (follow this order):**

1. **Header and imports:**
   ```julia
   using QuantumFurnace
   using LinearAlgebra
   using Statistics
   using Printf
   using LsqFit
   using Dates
   ```

2. **BLAS thread control:** Set `BLAS.set_num_threads(4)` at script start. Store original count for restore in finally block.

3. **Physical parameters:** Use the same locked parameters as test_helpers.jl:
   - `BETA = 10.0`, `SIGMA = 1.0 / BETA`
   - `NUM_ENERGY_BITS = 12`, `W0 = 0.05`
   - `T0 = 2pi / (2^NUM_ENERGY_BITS * W0)`, `NUM_TROTTER_STEPS_PER_T0 = 10`
   - `a = BETA / 30.0`, `b_metro = 0.4` (name `b_metro` to avoid shadowing with the scaling base `b`)

4. **System factory function `make_benchmark_system(n_qubits; trotter=nothing)`:**
   - Use `load_hamiltonian("heis", n_qubits; beta=BETA)` to load the Hamiltonian. This works in standalone scripts when run with `julia --project=@.` from the project root (uses `Pkg.project().path`).
   - Create jump operators: single-site Paulis (X, Y, Z) on each site, normalized by `sqrt(3 * n_qubits)`.
   - Select basis: `trotter.eigvecs` for TrotterDomain, `hamiltonian.eigvecs` otherwise.
   - Return `(; hamiltonian, jumps, n_qubits)`.

5. **Config factory functions:**
   - `make_bench_liouv_config(n_qubits, domain; with_coherent=true)` -- returns `LiouvConfig` with the locked parameters and `num_qubits=n_qubits`.

6. **krylovdim heuristic table:**
   ```julia
   KRYLOVDIM_HEURISTIC = Dict(3 => 30, 4 => 30, 5 => 30, 6 => 50, 7 => 50)
   ```
   The n=6 value is the initial probe value; it may be updated to 100 by the probe step.

7. **krylovdim probe function `probe_krylovdim_n6(config, hamiltonian, jumps; trotter=nothing)`:**
   - Build dense Lindbladian: `L_dense = construct_lindbladian(jumps, config, hamiltonian; trotter=trotter)`
   - Extract dense reference: `dense_result = extract_leading_eigendata(L_dense; n_modes=4)`
   - Try krylovdim=50: run `krylov_spectral_gap` and compute `abs(krylov_gap - dense_gap)`
   - If error < 1e-8, return 50
   - Otherwise try krylovdim=100: if error < 1e-8, return 100
   - Otherwise error with diagnostic message
   - Print the probe results to stdout

8. **Benchmark function `run_benchmark(config, hamiltonian, jumps; krylovdim, n_runs=5, trotter=nothing)`:**
   - Warmup: one untimed `krylov_spectral_gap` call (per-n JIT)
   - `GC.gc()` before warmup
   - Timed runs loop (n_runs iterations):
     - `GC.gc()` before each run
     - `t = @elapsed result = krylov_spectral_gap(config, hamiltonian, jumps; trotter=trotter, krylovdim=krylovdim, howmany=4)`
     - `a_bytes = @allocated krylov_spectral_gap(config, hamiltonian, jumps; trotter=trotter, krylovdim=krylovdim, howmany=4)` (separate call for allocation measurement)
     - Collect times, allocs, matvec counts, spectral gaps
   - Return `(; times, allocs, matvec_counts, gaps, median_time=median(times), median_allocs=median(allocs), result=last_result)`

9. **Power-law fit function `fit_scaling(n_values, t_values)`:**
   - Model: `model(n, p) = p[1] .* (p[2] .^ n)`
   - Initial guess: `p0 = [1e-3, 4.0]`
   - `fit = curve_fit(model, Float64.(n_values), t_values, p0)`
   - Extract `a_fit, b_fit = fit.param`
   - **Hard assertion:** `@assert 3.5 <= b_fit <= 4.5 "Scaling base b=$(b_fit) outside [3.5, 4.5]"`
   - Return `(; a=a_fit, b=b_fit, fit)`

10. **Report generation function `write_report(report_path, energy_data, trotter_data, energy_fit, trotter_fit, metadata)`:**
    - Generate markdown report with:
      - Header with date, Julia version, BLAS threads, system info
      - **EnergyDomain KMS section:** timing table (n, dim^2, krylovdim, median_time, alloc_GB, matvecs, spectral_gap), scaling fit (a, b)
      - **TrotterDomain KMS section:** same table format, scaling fit
      - **Extrapolation section:** predicted times and memory for n=10, n=12 for both domains, feasibility notes (EnergyDomain feasible at n=10, TrotterDomain likely infeasible due to NUFFT prefactors)
      - **Memory Guard Calibration section:** compare empirical @allocated vs pre-flight estimate `krylovdim * 4^n * 16 * 1.5`, recommend threshold adjustment if needed
    - Use `@sprintf` for formatted numbers
    - Separate sections for EnergyDomain and TrotterDomain (NOT side-by-side) per locked decision

11. **Main function `main()`:**
    - Print script header to stdout
    - Initialize results containers for EnergyDomain and TrotterDomain
    - **EnergyDomain benchmarks:**
      - For n in [3, 4, 5, 6]:
        - Construct system via `make_benchmark_system(n)`
        - Create config via `make_bench_liouv_config(n, EnergyDomain())`
        - At n=6: run krylovdim probe, update `KRYLOVDIM_HEURISTIC[6]` and `KRYLOVDIM_HEURISTIC[7]` with chosen value
        - Run benchmark with `n_runs = (n <= 6) ? 5 : 1`
        - Print summary row to stdout
        - Record `Sys.maxrss()` after each n
      - Fit scaling to n=3..6 data
      - Print fit results
    - **n=7 conditional (EnergyDomain):**
      - Predict t7 from fit: `a * b^7`
      - If t7 < 15*60 seconds: construct n=7 system, run single benchmark, print results
      - If t7 >= 15*60: print skip message
    - **TrotterDomain benchmarks:**
      - For n in [3, 4, 5, 6]:
        - Construct Trotter: `TrottTrott(hamiltonian, T0, NUM_TROTTER_STEPS_PER_T0)`
        - Reconstruct system with Trotter eigenbasis: `make_benchmark_system(n; trotter=trotter)`
        - Create config via `make_bench_liouv_config(n, TrotterDomain())`
        - At n=6: krylovdim probe against dense (same probe logic)
        - Run benchmark
        - Print summary row
      - Fit scaling
    - **n=7 conditional (TrotterDomain):**
      - Same logic as EnergyDomain
    - **Report generation:**
      - Create `results/` directory if it does not exist: `mkpath(joinpath(dirname(@__DIR__), "results"))`
      - Write report to `results/krylov_scaling_report.md`
      - Print "Report saved to: ..."
    - Wrap everything in try-finally to restore BLAS threads

12. **Script entry point:** Call `main()` at module level.

**Important implementation details:**
- Use `@elapsed` for timing, NOT BenchmarkTools (benchmarks take seconds to minutes).
- Call `GC.gc()` before each timed run per research pitfall guidance.
- Per-n warmup: for each n, one untimed call before timed repetitions.
- `@allocated` is measured in a SEPARATE call from the timed run (cannot combine `@elapsed` and `@allocated` on the same expression easily). Run it once after the timed runs.
- The n=6 krylovdim probe dense reference is NOT part of the timed benchmark.
- For TrotterDomain, the `trotter` keyword must be passed to both `make_benchmark_system` and `krylov_spectral_gap`.
- Use `load_hamiltonian("heis", n; beta=BETA)` which works when the script is invoked as `julia --project=@. simulations/main_krylov_benchmark.jl` from the project root.
- Print stdout summary tables during execution per locked decision.
