**Struct definition:**
```julia
struct KrylovWorkspace{T<:Complex}
    # Precomputed data (immutable after construction)
    precomputed_data::NamedTuple        # from _precompute_data()
    B_total::Union{Nothing, Matrix{T}}  # Precomputed coherent B (nothing for GNS / with_coherent=false)
    jumps::Vector{JumpOp}               # Reference to jump operators

    # Scratch matrices for dissipator accumulation (dim x dim)
    jump_oft::Matrix{T}       # A(omega) buffer (written by oft!)
    tmp1::Matrix{T}           # scratch for mul! results
    tmp2::Matrix{T}           # scratch for mul! results
    LdagL::Matrix{T}          # scratch for L'*L product
    rho_out::Matrix{T}        # output accumulator (zeroed at start of each matvec)
end
```

Per user decision: the workspace is designed for all domains from the start. Phase 28 adds dispatch methods, not workspace restructuring. The workspace is tied to (config, ham) at construction.

**Constructor signature:**
```julia
function KrylovWorkspace(
    config::AbstractLiouvConfig,
    hamiltonian::HamHam,
    jumps::Vector{JumpOp};
    trotter::Union{TrottTrott, Nothing}=nothing,
)
```

Constructor must:
1. Determine `ham_or_trott`: use `trotter` for TrotterDomain, `hamiltonian` otherwise (mirror `construct_lindbladian` pattern from `furnace.jl`)
2. Call `_precompute_data(config, ham_or_trott)` to get precomputed data (transition, gamma_norm_factor, energy_labels, and domain-specific fields)
3. Call `_precompute_coherent_total_B(jumps, ham_or_trott, config, precomputed_data)` to get B_total (returns nothing for GNS / with_coherent=false)
4. Get `dim = size(hamiltonian.data, 1)` and element type `CT = Complex{eltype(hamiltonian.eigvals)}`
5. Allocate 5 scratch matrices: `jump_oft`, `tmp1`, `tmp2`, `LdagL`, `rho_out` -- all `zeros(CT, dim, dim)`
6. Return the constructed `KrylovWorkspace{CT}`

Important: the `precomputed_data` is a NamedTuple whose fields vary by domain. Store it as `NamedTuple` (not `Any`). Use a type parameter if needed for type stability, but the simple approach is to use the concrete NamedTuple type returned by `_precompute_data`. Since the struct has `T<:Complex` as type parameter, and the NamedTuple type varies, use a second type parameter:
```julia
struct KrylovWorkspace{T<:Complex, PD<:NamedTuple}
    precomputed_data::PD
    ...
end
```

This ensures zero overhead when accessing precomputed_data fields.

**Helper: `_accumulate_dissipator!`**

Factored dissipator formula that both forward and adjoint call. Per user decision, the adjoint is achieved by passing `L_op'` where forward passes `L_op`:

```julia
function _accumulate_dissipator!(
    out::Matrix{<:Complex},
    L_op::AbstractMatrix{<:Complex},
    rho::AbstractMatrix{<:Complex},
    scalar::Real,
    ws,  # KrylovWorkspace -- uses tmp1, tmp2, LdagL
)
```

Computes: `out += scalar * (L_op * rho * L_op' - 0.5*(L_op' * L_op * rho + rho * L_op' * L_op))`

Implementation using BLAS `mul!` for zero allocations:
1. `mul!(ws.LdagL, L_op', L_op)` -- L'L
2. `mul!(ws.tmp1, L_op, rho)` -- L*rho
3. `mul!(ws.tmp2, ws.tmp1, L_op')` -- L*rho*L'
4. `@. out += scalar * ws.tmp2` -- accumulate sandwich term
5. `mul!(ws.tmp1, ws.LdagL, rho)` -- L'L*rho
6. `@. out -= (0.5 * scalar) * ws.tmp1` -- accumulate anticommutator term 1
7. `mul!(ws.tmp1, rho, ws.LdagL)` -- rho*L'L
8. `@. out -= (0.5 * scalar) * ws.tmp1` -- accumulate anticommutator term 2

CRITICAL: All `mul!` calls use 3-arg form. The `@.` broadcasting for accumulation is allocation-free for `Matrix` types. Do NOT use 5-arg `mul!` with complex alpha/beta for the accumulation -- the `@.` pattern is clearer and equally allocation-free.

**Forward: `apply_lindbladian!` for EnergyDomain**

```julia
function apply_lindbladian!(
    ws::KrylovWorkspace,
    rho::AbstractMatrix{<:Complex},
    config::AbstractLiouvConfig{EnergyDomain},
    hamiltonian::HamHam,
)
```

Per user decision: dispatch on domain type via config parametrization. Signature designed for KrylovKit closure wrapping: `rho -> (apply_lindbladian!(ws, rho, config, ham); copy(ws.rho_out))`.

Implementation mirrors `_jump_contribution!` for `AbstractLiouvConfig{EnergyDomain}` in `jump_workers.jl:47-89` line-by-line:

1. Extract from `ws.precomputed_data`: `transition`, `gamma_norm_factor`, `energy_labels`
2. `fill!(ws.rho_out, 0)` -- CRITICAL: zero output at each call
3. **Coherent term** (per user decision: read `with_coherent` from config, not keyword):
   - If `ws.B_total !== nothing`: compute `-i[B, rho]`
   - `mul!(ws.tmp1, ws.B_total, rho)` then `@. ws.rho_out += -1im * ws.tmp1`
   - `mul!(ws.tmp1, rho, ws.B_total)` then `@. ws.rho_out += 1im * ws.tmp1`
   - GNS configs have `with_coherent=false` so B_total is nothing -- silently skipped (per user decision)
4. **Dissipator loop**: exactly mirrors `jump_workers.jl:64-86`
   - `prefactor = (config.w0 / (config.sigma * sqrt(2 * pi))) * gamma_norm_factor`
   - For each jump in `ws.jumps`:
     - If `jump.hermitian`: iterate half-grid (`w_raw > 1e-12 && continue`, `w = abs(w_raw)`)
       - `oft!(ws.jump_oft, jump, w, hamiltonian, config.sigma)`
       - `scalar_w = prefactor * transition(w)`
       - `_accumulate_dissipator!(ws.rho_out, ws.jump_oft, rho, scalar_w, ws)`
       - If `w > 1e-12` (negative freq partner):
         - `scalar_neg = prefactor * transition(-w)`
         - `_accumulate_dissipator!(ws.rho_out, ws.jump_oft', rho, scalar_neg, ws)` -- lazy `'` per user decision
     - Else (non-Hermitian): iterate all energy_labels
       - `oft!(ws.jump_oft, jump, w, hamiltonian, config.sigma)`
       - `_accumulate_dissipator!(ws.rho_out, ws.jump_oft, rho, scalar_w, ws)`
5. `return ws.rho_out`

**Adjoint: `apply_adjoint_lindbladian!` for EnergyDomain**

Per user decision: separate function (not keyword flag), shared KrylovWorkspace.

```julia
function apply_adjoint_lindbladian!(
    ws::KrylovWorkspace,
    rho::AbstractMatrix{<:Complex},
    config::AbstractLiouvConfig{EnergyDomain},
    hamiltonian::HamHam,
)
```

Differences from forward (per research Adjoint Lindbladian section):
1. Coherent term: `+i[B, rho]` instead of `-i[B, rho]` (sign flip)
   - `@. ws.rho_out += 1im * ws.tmp1` (B*rho term, was -1im)
   - `@. ws.rho_out -= 1im * ws.tmp1` (rho*B term, was +1im)
2. Dissipator: swap L <-> L' in the call to `_accumulate_dissipator!`
   - Where forward passes `ws.jump_oft`, adjoint passes `ws.jump_oft'`
   - Where forward passes `ws.jump_oft'` (negative freq), adjoint passes `ws.jump_oft`
   - The `_accumulate_dissipator!` helper handles the rest automatically

The loop structure is identical to forward -- only the L_op argument to `_accumulate_dissipator!` is swapped.
