<objective>
Close the verification gap for Phase 24 Truth 3: the validation script must demonstrate gap estimation agreement for n=4 and n=6 Heisenberg chains within a defined, checked tolerance.

Purpose: The trajectory step applies `delta_eff = delta * n_jumps` worth of channel evolution per step, but the time axis labels steps as `step * delta`. This means the fitted decay rate is `n_jumps` times faster than the continuous-time Liouvillian spectral gap. The script currently reports this factor but still fails its own pass criterion (`within_ci || relative_error < 0.3` with relative_error ~19-27). This plan applies the normalization correction so the script demonstrates actual agreement and outputs PASS.

Output: Updated validation script that normalizes the fitted gap by `n_jumps` before cross-validation, checks normalized agreement, and outputs PASS for both system sizes.
</objective>
