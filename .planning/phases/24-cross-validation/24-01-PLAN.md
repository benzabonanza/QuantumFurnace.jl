**CrossValidationResult struct (concrete types, no type parameter per Phase 23 Aqua decision):**
```julia
struct CrossValidationResult
    fitted_gap::Float64
    exact_gap::Float64
    relative_error::Float64
    absolute_error::Float64
    within_ci::Bool
    imaginary_ratio::Float64
    imaginary_warning::Bool
end
```

**cross_validate_gap function -- two methods via dispatch:**

Method 1: `cross_validate_gap(estimated::SpectralGapResult, exact_result::LindbladianResult)` -- extracts `exact_result.spectral_gap` and forwards to Method 2.

Method 2: `cross_validate_gap(estimated::SpectralGapResult, exact_eigenvalue::Complex)` -- core implementation:
- `exact_gap = abs(real(exact_eigenvalue))` (LOCKED DECISION -- not `abs(exact_eigenvalue)`)
- `fitted_gap = estimated.gap`
- `relative_error = abs(fitted_gap - exact_gap) / exact_gap`
- `absolute_error = abs(fitted_gap - exact_gap)`
- `within_ci = estimated.gap_ci[1] <= exact_gap <= estimated.gap_ci[2]`
- `im_ratio = abs(imag(exact_eigenvalue)) / abs(real(exact_eigenvalue))`
- `im_warning = im_ratio > 0.1`
- If `im_warning`, emit `@warn` with message: "Exact eigenvalue has significant imaginary part (|Im/Re| = $(round(im_ratio; digits=3))). Pure exponential fit may not capture oscillatory decay." with `exact_eigenvalue` kwarg.
- Return `CrossValidationResult(fitted_gap, exact_gap, relative_error, absolute_error, within_ci, im_ratio, im_warning)`

**Handle edge case:** If `real(exact_eigenvalue) == 0.0`, set `im_ratio = Inf` and `relative_error = Inf` to avoid division by zero.
