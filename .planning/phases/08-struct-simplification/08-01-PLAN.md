1. Create a `_build_common_fields()` helper function that returns a NamedTuple of the shared fields with proper defaults. The shared fields across all 4 config types are:
   - `num_qubits::Int64` (required)
   - `with_coherent::Bool` (required, but GNS enforces false)
   - `with_linear_combination::Bool` (required)
   - `domain::D` (required, carries the type param)
   - `beta::Float64` (required)
   - `sigma::Float64` (required)
   - `gaussian_parameters::Union{Tuple{Float64, Float64}, Tuple{Nothing, Nothing}} = (nothing, nothing)`
   - `a::Union{Float64, Nothing} = nothing`
   - `b::Union{Float64, Nothing} = nothing`
   - `num_energy_bits::Union{Int, Nothing} = nothing` (was Int64 = -1)
   - `t0::Union{Float64, Nothing} = nothing` (was Float64 = -1.)
   - `w0::Union{Float64, Nothing} = nothing` (was Float64 = -1.)
   - `eta::Union{Float64, Nothing} = nothing` (was Float64 = -1.)
   - `num_trotter_steps_per_t0::Union{Int, Nothing} = nothing` (was Int64 = -1)

   Implementation approach: Since Julia @kwdef structs can't call helper functions in their field defaults, use a different strategy. Define the 4 @kwdef structs with the correct Union{..., Nothing} field types directly but WITHOUT duplication by using a macro or simply writing them cleanly with the updated types. The `_build_common_fields` decision means: extract a function that each constructor's inner constructor can call to validate/transform common fields. However, the simplest approach honoring `@kwdef` is: define each struct with the Union types directly (the "deduplication" is conceptual -- the fields are still listed, but defaults are `nothing` instead of `-1`), and add an inner constructor that enforces GNS `with_coherent=false`.

   Actually, implement `_build_common_fields` as a function that receives all common kwargs and returns them as a NamedTuple. Then define custom outer constructors for each config type that call `_build_common_fields` and pass results to the default constructor. This way:
   - Each `@kwdef` struct still lists its fields (Julia requires this)
   - But the construction logic is shared via `_build_common_fields`
   - GNS variants' constructor enforces `with_coherent=false` by overriding the value

   Concrete implementation:
   ```julia
   function _build_common_fields(;
       num_qubits::Int64,
       with_coherent::Bool,
       with_linear_combination::Bool,
       beta::Float64,
       sigma::Float64,
       gaussian_parameters::Union{Tuple{Float64, Float64}, Tuple{Nothing, Nothing}} = (nothing, nothing),
       a::Union{Float64, Nothing} = nothing,
       b::Union{Float64, Nothing} = nothing,
       num_energy_bits::Union{Int, Nothing} = nothing,
       t0::Union{Float64, Nothing} = nothing,
       w0::Union{Float64, Nothing} = nothing,
       eta::Union{Float64, Nothing} = nothing,
       num_trotter_steps_per_t0::Union{Int, Nothing} = nothing,
   )
       return (; num_qubits, with_coherent, with_linear_combination,
               beta, sigma, gaussian_parameters, a, b,
               num_energy_bits, t0, w0, eta, num_trotter_steps_per_t0)
   end
   ```

   ```julia
       new{D}(...all fields...)
   end
   ```
   Note: Since @kwdef generates an outer constructor that feeds into the inner, the inner constructor enforcement is the right pattern.
