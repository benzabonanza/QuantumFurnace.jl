<objective>
Implement `run_observable_trajectories` -- a trajectory runner that measures time-resolved observables without per-trajectory density matrix reconstruction, with optional end-of-run DM reconstruction via `reconstruct_dm=true`.

Purpose: Phase 23 (Gap Estimation API) needs time-resolved observable decay curves from trajectories. The existing `run_trajectories` always computes `rho_mean`, which is unnecessary overhead when only observable time series are needed. This dedicated function skips DM accumulation by default, reducing per-trajectory cost for larger systems.

Output: Working `run_observable_trajectories` function, `ObservableTrajectoryResult` struct, cross-validation tests proving bitwise agreement with existing runner.
</objective>

**3. Add `run_observable_trajectories` to src/trajectories.jl** (after `run_trajectories`, around line 640):

```julia
"""
    run_observable_trajectories(jumps, config, psi0, hamiltonian;
        trotter=nothing, total_time=config.mixing_time, delta=config.delta,
        ntraj=1, observables, save_every=1, seed=nothing, reconstruct_dm=false)

Run trajectory simulations measuring time-resolved observables without per-trajectory
density matrix reconstruction. When `reconstruct_dm=true`, also accumulates the averaged
density matrix (using the existing `_run_chunk_with_obs!` path).

Returns an `ObservableTrajectoryResult` with `rho_mean=nothing` when `reconstruct_dm=false`.
"""
```

Implementation logic:
- Call `_build_framework_and_seed(jumps, config, psi0, hamiltonian; trotter, delta, seed)` for one-time setup
- Compute `num_steps = ceil(Int, total_time / delta_step)`, `num_saves = div(num_steps, save_every) + 1`, `num_obs = length(observables)` -- MUST match existing run_trajectories (lines 576-578)
- Build time grid: `times[s] = (s-1) * save_every * delta_step` for s in 1:num_saves (same as lines 580-583)
- **Multi-threaded path** (ntraj > 1 && Threads.nthreads() > 1):
  - Partition with `_partition_trajectories(1:ntraj, nt)`
  - Create per-task workspaces and mean_data arrays
  - BLAS thread save/restore with try/finally
  - If `reconstruct_dm`: use `_run_chunk_with_obs!` (existing function, accumulates DM in ws.rho_acc)
  - If NOT `reconstruct_dm`: use `_run_chunk_obs_only!` (new function, no DM accumulation)
  - Sum mean_data_per_task, divide by ntraj
  - If `reconstruct_dm`: sum ws.rho_acc across tasks, divide by ntraj, hermitianize!
- **Serial path** (else):
  - Single workspace, single mean_data matrix
  - If `reconstruct_dm`: inline loop with both `_accumulate_measurements!` AND `_accumulate_density_matrix!` (same as existing serial observable path lines 610-636)
  - If NOT `reconstruct_dm`: inline loop with only `_accumulate_measurements!` (no DM call)
  - Divide mean_data by ntraj
  - If `reconstruct_dm`: divide rho_acc by ntraj, hermitianize!
- Return `ObservableTrajectoryResult{CT}(times, mean_data, ntraj, actual_seed, rho_mean_or_nothing)`

CRITICAL: The seed-to-trajectory mapping `Xoshiro(master_seed + traj_id)` and the step/measurement loop must be identical to `run_trajectories` and `_run_chunk_with_obs!` to guarantee bitwise cross-validation.
