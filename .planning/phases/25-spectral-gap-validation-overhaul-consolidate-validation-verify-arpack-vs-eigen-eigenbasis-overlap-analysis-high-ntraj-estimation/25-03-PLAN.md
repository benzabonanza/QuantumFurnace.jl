    **Script sections:**

    **Section 0: Setup**
    ```julia
    using QuantumFurnace
    using LinearAlgebra

    const BETA = 10.0

    const NTRAJ = 20_000
    const SAVE_EVERY = 10

    ```

    **Section 1: ARPACK vs eigen verification (n=4 only)**
    - Create Heisenberg chain: `ham = HamHam([[X,X],[Y,Y],[Z,Z]], [1.0,1.0,1.0], 4, BETA; periodic=true)`
    - Create config: `config = ThermalizeConfig(TimeDomain(), num_qubits=4, ...)`
      Actually, use the standard config creation pattern from the codebase. Check how validate_gap_estimation.jl created configs. The pattern is:
      ```julia
      config = LiouvConfig(TimeDomain(); num_qubits=n, beta=BETA, delta=DELTA, with_coherent=false)
      ```
      Then construct the Lindbladian:
      ```julia
      liouv_result = run_lindbladian(jumps, config, ham)
      L = liouv_result.liouvillian
      ```
    - Compute exact gap via ARPACK (from run_lindbladian): `gap_arpack = abs(real(liouv_result.spectral_gap))`
    - Compute exact gap via dense eigen:
      ```julia
      F = eigen(Matrix(L))
      sorted_idx = sortperm(abs.(real.(F.values)))
      gap_eigen = abs(real(F.values[sorted_idx[2]]))
      ```
    - Print comparison:
      ```
      === ARPACK vs eigen verification (n=4) ===
      ARPACK gap: 0.XXXXX
      eigen  gap: 0.XXXXX
      Difference: X.XXe-XX
      PASS/FAIL (threshold: 1e-8)
      ```
    - Use `abs(gap_arpack - gap_eigen) < 1e-8` as pass criterion.

    **Section 2: For each system size (n=4, n=6):**

    a. **System setup:**
       - Create Hamiltonian, jumps (via pick_transition + create_alpha or whatever pattern the codebase uses to create JumpOp vectors -- check how existing validation scripts or tests create jumps). The standard pattern from the codebase is:

         Actually, `run_lindbladian` takes jumps as input. The jumps are created separately. Check how test_helpers.jl creates jumps. The pattern is: the jumps come from the Lindbladian construction. Look at how `construct_lindbladian` works -- it takes the same args and returns jumps.

         The correct pattern from the existing validate_gap_estimation.jl (now deleted, but we read it in research):

         But we need the jumps first. The jumps come from the Lindbladian construction.          Look at the test_helpers pattern. The test fixtures create jumps via `pick_transition` and `create_alpha`. The validation script should follow the same pattern. Read `src/furnace.jl` to understand `run_lindbladian`'s signature.

         1. Create `ham = HamHam(...)` with system parameters
         2. Create `alpha = create_alpha(ham, ...)` for KMS (or `create_alpha_gns` for GNS)

         Follow whatever pattern test_helpers.jl uses. The executor should read test_helpers.jl to confirm.
