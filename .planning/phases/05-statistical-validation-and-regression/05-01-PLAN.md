3. **Imports:** `using Test, LinearAlgebra, Random`. Include `test_helpers.jl` via `include(joinpath(@__DIR__, "..", "test_helpers.jl"))`.

4. **Convergence test helper function:** Create `convergence_ratio_test(domain; with_coherent, delta=0.1)` that:
   - Computes DM reference via `exp(delta * L) * vec(rho0)` (reuse single_step_crossval pattern from Phase 4)
   - Uses `psi0 = fill(ComplexF64(1.0), SMALL_DIM) / sqrt(SMALL_DIM)` as initial state
   - Builds trajectory framework once (same as Phase 4: `make_small_thermalize_config`, `precompute_data`, `KrausScratch`, `build_trajectoryframework`)
   - For TrotterDomain: pass `trotter=SMALL_TROTTER` to `construct_lindbladian` and use `SMALL_TROTTER` as `ham_or_trott`
   - Runs N_traj geometric progression: `[1_000, 4_000, 16_000, 64_000]` (4 points, factor-of-4 steps)
   - For each N_traj point: seed with `Random.seed!(42)`, run N trajectories via `step_along_trajectory!`, accumulate `psi * psi'`, average, Hermitianize, compute `trace_distance_h(Hermitian(rho_dm), Hermitian(rho_traj))`
   - **Key:** Use same seed (42) for all N_traj points so smaller runs are a prefix of larger runs (reduces ratio variance)
   - Returns `(errors, ratios)` where `ratios[i] = errors[i] / errors[i+1]`

5. **TVAL-05 testsets:**
   - `@testset "TVAL-05: EnergyDomain 1/sqrt(N) convergence"`: Call `convergence_ratio_test(EnergyDomain(); with_coherent=false)`. Assert each ratio in `[1.5, 2.5]`. Print errors and ratios as diagnostics.
   - `@testset "TVAL-05: TrotterDomain 1/sqrt(N) convergence"`: Call `convergence_ratio_test(TrotterDomain(); with_coherent=true)`. Assert each ratio in `[1.5, 2.5]`. Print errors and ratios as diagnostics.

**Pitfall avoidance:**
- Use `Random.seed!(42)`, NOT StableRNG (trajectory code uses global `rand()`)
- Compare trajectory against DM result (NOT Gibbs state) -- isolates sampling convergence from domain approximation
- delta=0.1 ensures O(delta^2) systematic error (~10^-4) is negligible compared to 1/sqrt(1000) statistical error (~0.01)
- Save plain `Matrix{ComplexF64}`, not `Hermitian` wrappers in intermediate computations
