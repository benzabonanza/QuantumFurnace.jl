**2. Observable builder: `build_convergence_observables`**
```julia
function build_convergence_observables(hamiltonian::HamHam, num_qubits::Int)
```
- Build nearest-neighbor Z_iZ_{i+1} correlation matrices using `pad_term([Z, Z], num_qubits, i; periodic=true)`.
- Transform each ZZ matrix from computational basis to eigenbasis: `ZZ_eigen = V' * ZZ_comp * V` where `V = hamiltonian.eigvecs`.
- Build energy observable: diagonal matrix of `hamiltonian.eigvals` in eigenbasis (use `diagm(hamiltonian.eigvals)` cast to ComplexF64).
- Return `(observables::Vector{Matrix{ComplexF64}}, names::Vector{String})`.
- Names: `"ZZ_12"`, `"ZZ_23"`, ..., `"ZZ_N1"` (periodic) and `"H"`.

CRITICAL BASIS NOTE: All observables MUST be in the Hamiltonian eigenbasis because `run_trajectories` returns `rho_mean` in the eigenbasis for EnergyDomain (the state vector psi lives in the eigenbasis). The Gibbs state `hamiltonian.gibbs` is also in eigenbasis. Using computational basis observables would give wrong `tr(rho * O)` values. See Research Pitfall 1.

**3. TrotterDomain support: `build_convergence_observables_trotter`**
```julia
function build_convergence_observables_trotter(hamiltonian::HamHam, trotter::TrottTrott, num_qubits::Int)
```
Same as above, but transforms observables to Trotter eigenbasis instead:
`O_trotter = V_T' * V_H * V_H' * O_comp * V_H * V_H' * V_T` simplifies to `O_trotter = V_T' * O_comp * V_T` where `V_T = trotter.eigvecs`.
Wait -- more carefully: `O_comp` in computational basis. To get to Trotter eigenbasis: `O_trotter = V_T' * O_comp * V_T`. The H observable in Trotter basis is `V_T' * hamiltonian.data * V_T` (NOT diagonal).

Also add a helper to get the Gibbs state in Trotter basis (following furnace.jl line 116-118):
```julia
function _gibbs_in_trotter_basis(hamiltonian::HamHam, trotter::TrottTrott)
    return Hermitian(trotter.eigvecs' * hamiltonian.eigvecs * hamiltonian.gibbs *
                     hamiltonian.eigvecs' * trotter.eigvecs)
end
```

**4. Gibbs observable value computation:**
```julia
function _compute_gibbs_observable_values(gibbs::Hermitian, observables::Vector{<:Matrix{<:Complex}})
    return [real(tr(Matrix(gibbs) * obs)) for obs in observables]
end
```

**5. Main function: `run_trajectories_convergence`**
```julia
function run_trajectories_convergence(
    jumps::Vector{JumpOp},
    config::AbstractThermalizeConfig,
    psi0::Vector{<:Complex},
    hamiltonian::HamHam;
    gibbs::Hermitian,
    observables::Vector{<:Matrix{<:Complex}},
    observable_names::Vector{String},
    batch_size::Int = 1000,
    n_batches::Int = 10,
    seed::Union{Int,Nothing} = nothing,
    trotter::Union{TrottTrott,Nothing} = nothing,
    total_time::Real = config.mixing_time,
    delta::Real = config.delta,
)
```

Implementation details:
- Generate `actual_seed` from `Random.RandomDevice()` if seed is nothing (same pattern as `run_trajectories`).
- Pre-allocate `rho_acc = zeros(CT, dim, dim)` for running sum accumulation.
- Pre-compute `obs_gibbs = _compute_gibbs_observable_values(gibbs, observables)`.
- Pre-allocate storage vectors: `trace_dists`, `obs_values_list` (Vector of Vector{Float64}, horizontally concat at end to Matrix), `cum_n_traj`, `batch_sizes_vec`.
- Loop over `1:n_batches`:
  - Compute `batch_seed = actual_seed + n_total` (ensures non-overlapping trajectory seed sequences).
  - Call existing `run_trajectories(jumps, config, psi0, hamiltonian; trotter=trotter, total_time=total_time, delta=delta, ntraj=batch_size, seed=batch_seed)`.
  - Accumulate: `rho_acc .+= result.rho_mean .* batch_size`.
  - Update `n_total += batch_size`.
  - Compute running average: `rho_running = rho_acc ./ n_total`, then `hermitianize!(rho_running)`.
  - Measure trace distance: `trace_distance_h(Hermitian(rho_running), gibbs)`.
  - Measure observables: `[real(tr(rho_running * obs)) for obs in observables]`.
  - Push metrics to storage vectors.
- After loop: construct final `rho_final = rho_acc ./ n_total; hermitianize!(rho_final)`.
- Build `ConvergenceData` from accumulated vectors. Convert `obs_values_list` to Matrix via `hcat(obs_values_list...)` or pre-allocate Matrix.
- Build `TrajectoryResult(rho_final, n_total, actual_seed, nothing, nothing)`.
- Return `(TrajectoryResult, ConvergenceData)` tuple.

IMPORTANT: Do NOT pass observables to run_trajectories -- that parameter triggers time-resolved measurement tracking WITHIN each trajectory, which is different from batch-level convergence monitoring. We pass `observables=nothing` to run_trajectories and compute `tr(rho * O)` ourselves from the running average.

```julia
# ---------------------------------------------------------------------------
# ConvergenceData <-> Dict conversion (for safe BSON serialization)
# ---------------------------------------------------------------------------

function _convergence_to_dict(conv::ConvergenceData)
    return Dict{Symbol, Any}(
        :batch_sizes           => conv.batch_sizes,
        :cumulative_n_traj     => conv.cumulative_n_traj,
        :trace_distances       => conv.trace_distances,
        :observable_names      => conv.observable_names,
        :observable_values     => conv.observable_values,
        :observable_gibbs_values => conv.observable_gibbs_values,
    )
end

function _dict_to_convergence(d::Dict)
    return ConvergenceData(
        d[:batch_sizes],
        d[:cumulative_n_traj],
        d[:trace_distances],
        d[:observable_names],
        d[:observable_values],
        get(d, :observable_gibbs_values, Float64[]),
    )
end
```

These follow the exact same Dict-based serialization pattern as `_experiment_to_dict` / `_dict_to_experiment`. The `get` with default in `_dict_to_convergence` provides forward compatibility (if loading a file saved before `observable_gibbs_values` was added).
