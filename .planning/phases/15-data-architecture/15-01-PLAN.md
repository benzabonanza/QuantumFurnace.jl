Create `src/results.jl` with the following components:

**1. ExperimentResult struct** (per locked decision):
```julia
struct ExperimentResult{C<:AbstractConfig, T<:AbstractFloat}
    config::C
    trajectory_result::TrajectoryResult{Complex{T}}
    hamiltonian_params::Dict{Symbol, Any}
    metadata::Dict{Symbol, Any}
end
```
Embed TrajectoryResult as a field (not flattened) per discretion recommendation.

**2. _experiment_to_dict(result::ExperimentResult) -> Dict{Symbol,Any}**
Convert ExperimentResult to a plain Dict for BSON serialization:
- Config: convert to sub-Dict with all fields as scalars/nothing. Store `config_type` as string ("KMS" or "GNS") based on whether config `isa Union{LiouvConfigGNS, ThermalizeConfigGNS}`. Store `domain` as `string(typeof(config.domain))` (e.g., "TrotterDomain"). Store `config_kind` as string ("liouv" or "thermalize") based on whether config `isa AbstractThermalizeConfig`. Include all config fields by name (num_qubits, with_coherent, with_linear_combination, beta, sigma, gaussian_parameters, a, b, num_energy_bits, t0, w0, eta, num_trotter_steps_per_t0, and mixing_time/delta if thermalize config).
- Trajectory: convert to sub-Dict with `:rho_mean => Matrix(result.trajectory_result.rho_mean)` (unwrap any Hermitian), `:n_trajectories`, `:seed`, `:times`, `:measurements_mean`.
- hamiltonian_params: store as-is (already a Dict{Symbol,Any}).
- metadata: store as-is (already a Dict{Symbol,Any}).

**3. _dict_to_experiment(d::Dict) -> ExperimentResult**
Reconstruct from Dict:
- Config: use `_reconstruct_config(d[:config])` helper.
- TrajectoryResult: reconstruct from `:trajectory` sub-Dict using `get()` for optional fields (times, measurements_mean) for forward compat.
- hamiltonian_params and metadata: use `get(d, key, Dict{Symbol,Any}())` for forward compat.
- Infer `T` from `eltype(real(traj.rho_mean))` and `C` from `typeof(config)`.

**4. _reconstruct_config(d::Dict) -> AbstractConfig**
- Use DOMAIN_LOOKUP Dict to convert string back to domain singleton.
- Use `config_type` ("KMS"/"GNS") and `config_kind` ("liouv"/"thermalize") to pick the right constructor.
- Build kwargs Dict from the stored fields, filtering out nothing values where appropriate.
- Use the @kwdef constructors: LiouvConfig, LiouvConfigGNS, ThermalizeConfig, ThermalizeConfigGNS.

**5. DOMAIN_LOOKUP constant:**
```julia
const DOMAIN_LOOKUP = Dict(
    "TrotterDomain" => TrotterDomain(),
    "TimeDomain" => TimeDomain(),
    "EnergyDomain" => EnergyDomain(),
    "BohrDomain" => BohrDomain(),
)
```

**Important patterns to follow:**
- All BSON serialization via Dict (no direct struct serialization) -- per research recommendation and proven codebase pattern (see generate_references.jl).
- Unwrap Hermitian to Matrix before saving (per pitfall 2 in research).
- Store domain as string, not singleton struct (per pitfall 3).
- Use `get()` with defaults for forward compatibility (per locked decision: missing fields fill with nothing/defaults).

**2. Hamiltonian parameter extraction:**

Do NOT store eigendecomposition, bohr_freqs, bohr_dict, or gibbs (those are derived quantities).

- Return `path`.

**5. Companion text file writer:**

- Config section: type (KMS/GNS), domain, num_qubits, beta, sigma, and mixing_time/delta if thermalize config
- Results section: N_traj, seed, threads, wall time, rho dimensions

**6. Filename generation:**
```julia
function _generate_experiment_filename(config::AbstractConfig)
```
Per locked decision format: `{db_str}_{n_str}_{beta_str}_{domain_str}_{date_str}.bson`

- domain_str: lowercase domain name without "Domain" suffix (e.g., "trotter")
- n_str: "n{num_qubits}"
- beta_str: "beta{Int(config.beta)}" (use `round(Int, config.beta)` for safety)
- date_str: `Dates.format(Dates.now(), dateformat"yyyymmdd")`
