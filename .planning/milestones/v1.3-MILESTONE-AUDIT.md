## Tech Debt

### Phase 25: API Simplification Trade-offs
- `CrossValidationResult` and `cross_validate_gap` removed — the dedicated convenience wrapper was replaced by `eigenbasis_overlap_analysis` which provides exact gap and overlap metrics. Users compare values manually.
- Imaginary-part `@warn` removed — eigenvalues returned in `OverlapAnalysisResult` for user inspection. Could be re-added as a utility if needed.

### Physics Limitations (not code defects)
- n=6 Heisenberg chain gap estimation fails the <1% target (10.7% error) because all 8 preset observables have zero overlap with the gap mode. This is a translational symmetry protection effect confirmed by Quick-25/26/27 investigations.
- Non-monotonic gap estimation error at small delta (Quick-30/31/32) originates in single-exponential fitting, not trajectory simulation. 7/8 observable errors decrease monotonically with delta. Multi-exponential or Prony methods are potential future improvements.
