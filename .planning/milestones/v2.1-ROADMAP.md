## Phase Details

### Phase 39: Per-Jump Precomputation
**Goal**: Eliminate per-step eigendecomposition from run_thermalize hot loop
**Requirements**: PRECOMP-01 through PRECOMP-04
**Plans**: 39-01 (precomputation infrastructure), 39-02 (integration and validation)

### Phase 40: Save Every
**Goal**: Control trace distance computation frequency via save_every keyword
**Requirements**: SAVE-01, SAVE-02
**Plans**: 40-01 (save_every keyword with gated observation)

### Phase 41: Threading
**Goal**: Multi-threaded BLAS + omega-loop parallelism for DM thermalization
**Requirements**: THREAD-01 through THREAD-05
**Plans**: 41-01 (BLAS thread management), 41-02 (omega-loop parallelism)

### Phase 42: Mixing Time Estimation
**Goal**: Exponential fit on trace distance curve with extrapolation and quality gates
**Requirements**: MIX-01 through MIX-08
**Plans**: 42-01 (LsqFit.jl + MixingTimeEstimate), 42-02 (comprehensive tests)

### Phase 43: Bi-Exponential Fitting
**Goal**: Reduce extrapolation error from ~26% to <5% via bi-exponential decay model
**Plans**: PLAN.md (BiexpFitResult, fit_biexponential_decay, model=:biexp)

## Key Decisions

- _precompute_per_jump_channels stores K0s/U_residuals only (no Rs) -- DM path doesn't need raw R
- BohrDomain _precompute_R uses precomputed bohr_is/bohr_js with fallback to hamiltonian.bohr_dict
- save_every gates observation only; physics runs every step unconditionally
- DM BLAS pattern: save → set_num_threads(nthreads) → try loop finally restore
- OMEGA_THREAD_THRESHOLD=50, per-task ThermalizeScratch, BLAS=1 during Julia-level parallelism
- MixingTimeEstimate is separate struct (not ThermalizeResults modification) -- BSON compatibility
- Explicit :biexp model keyword only (no :auto mode with AICc)
- Bi-exp extrapolation via Roots.Bisection (no closed-form for multi-exponential)
