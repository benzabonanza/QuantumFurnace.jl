# Requirements: QuantumFurnace.jl

**Defined:** 2026-02-16
**Core Value:** Correct and efficient classical simulation of Lindbladian-based quantum Gibbs samplers

## v1.3 Requirements

Requirements for milestone v1.3 Mixing Time Estimation. Each maps to roadmap phases.

### Observables

- [x] **OBS-01**: User can build total magnetization observable (M_z = sum Z_i) in Hamiltonian eigenbasis -- via `build_preset_trajectory_observables` (Phase 25 consolidation)
- [x] **OBS-02**: User can build total magnetization observable in Trotter eigenbasis (for TrotterDomain) -- via `build_preset_trajectory_observables(ham, n; trotter=trotter)`
- [x] **OBS-03**: User can build combined gap estimation observables (H, M_z, all ZZ correlations) via single function call -- 8-observable bundle: H, Mz, XX_avg, YY_avg, ZZ_avg, Mz_stagg, Z1, XZ_stagg

### Trajectory Runner

- [x] **TRAJ-01**: User can run trajectories measuring time-resolved observables without mid-simulation DM reconstruction -- `run_observable_trajectories` with `reconstruct_dm=false`
- [x] **TRAJ-02**: User can optionally reconstruct the averaged DM once at the end of the observable-only run -- `reconstruct_dm=true`
- [x] **TRAJ-03**: Observable-only runner supports multi-threaded execution (same pattern as existing run_trajectories) -- bitwise cross-validated

### Fitting

- [x] **FIT-01**: User can fit single-exponential decay model (A * exp(-gap * t) + C) to observable time series using LsqFit.jl -- `fit_exponential_decay`
- [x] **FIT-02**: Fitting auto-generates initial guess via log-linear estimate (no manual tuning required for typical data) -- `_log_linear_initial_guess` helper
- [x] **FIT-03**: User can specify fitting window (skip_initial fraction) to exclude early multi-exponential transient -- measurably improves fit
- [x] **FIT-04**: Fit returns quality metrics: R-squared, confidence interval on gap, standard error -- all fields in FitResult
- [x] **FIT-05**: Fit enforces gap > 0 via parameter bounds -- `lower=[−Inf, 0.0, −Inf]`

### Gap Estimation API

- [x] **GAP-01**: User can call `estimate_spectral_gap` to run trajectories + fit all observables + select best estimate -- 12-field SpectralGapResult
- [x] **GAP-02**: Result struct `SpectralGapResult` contains gap estimate, CI, per-observable gaps, R-squared values, and fit metadata -- all 12 fields verified
- [x] **GAP-03**: Best observable selected by fit quality (R-squared, convergence, gap > 0) -- smallest gap among converged fits

### Cross-Validation

- [x] **VAL-01**: User can cross-validate fitted gap against exact Liouvillian spectral gap using `abs(real(spectral_gap))` -- via `eigenbasis_overlap_analysis` + manual comparison (dedicated wrapper removed in Phase 25)
- [x] **VAL-02**: Cross-validation warns when imaginary part of exact eigenvalue is significant (|Im/Re| > 0.1) -- eigenvalues returned in OverlapAnalysisResult for user inspection (dedicated warning removed in Phase 25)
- [x] **VAL-03**: Validation script demonstrates gap estimation agreement for n=4 and n=6 -- n=4 PASS (0.72%), n=6 FAIL with physics diagnostic

## Out of Scope

Explicitly excluded. Documented to prevent scope creep.

| Feature | Reason |
|---------|--------|
| Prony's method for multi-exponential extraction | Severely ill-conditioned for noisy trajectory data; single-exponential with window selection is more robust |
| Full Liouvillian construction for n>6 | 65536x65536 at n=8; trajectory-based estimation is the whole point |
| Autocorrelation-based gap estimation | Requires stationarity; our trajectories start far from equilibrium |
| Stochastic trace estimation for Liouvillian gap | Requires Liouvillian matvec primitive we don't have |
| GPU-accelerated fitting | Fitting is tiny computation; trajectory generation is the bottleneck |
| Real-time plotting during fitting | Post-process from saved data; paper plots are separate |
