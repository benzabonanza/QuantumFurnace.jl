# X+ZZ-disordered 1D Heisenberg fixtures (qf-e4z.27)

**Fact.** `scripts/scratch_multiseed_disordered_fixtures.jl` now
generates a SECOND 1D Heisenberg fixture family with `[[X], [Z, Z]]`
disorder, in parallel to the legacy `[[Z], [Z, Z]]` family. File naming:

* Legacy: `heis_xxx_disordered_periodic_n{n}_seed{seed}.bson`
* New (qf-e4z.27): `heis_xxx_XZdisordered_periodic_n{n}_seed{seed}.bson`

Both families are kept on disk.

**Why:** The X-disorder term anti-commutes with the global Z-parity
operator `P = Z^⊗N` on each site it's placed on, so the disordered
Hamiltonian has `[P, H] ≠ 0` exactly — no Z^⊗N symmetry to project
into. This eliminates the entire parity-trap class that motivated
qf-e4z.26 and qf-e4z.27:

* Pre-qf-e4z.27, `predict_lindbladian_trajectory`'s seed
  `vec(rho_0) = vec(I/d)` was P̂-EVEN, so the Krylov subspace stayed
  P̂-even-only on the legacy Z+ZZ fixtures (which preserved
  `P = Z^⊗N` exactly because all disorder terms are Z-type). The
  reported `spectral_gap` was the P̂-EVEN sub-spectrum gap, not the
  true Lindbladian gap.
* On the new X+ZZ fixtures the Hamiltonian has no exact Z^⊗N
  symmetry, so the trajectory predictor's `spectral_gap` and
  `krylov_spectral_gap`'s gap AGREE without needing any seed-
  perturbation trick. (The qf-e4z.27 two-pass architecture
  [[predict-trajectory-two-pass-qf-e4z-27]] is still applied because
  it's the universal correct default, but on these fixtures it's a
  no-op correctness-wise.)

**How to apply:**
- New numerical work that wants to AVOID the parity-trap interpretive
  layer: load the X+ZZ family. The "even-n collapses, odd-n flat"
  narrative from qf-e4z.23 (which is itself a parity-EVEN-gap-from-I/d
  artefact on Z+ZZ fixtures) should NOT carry over — X+ZZ produces a
  generic disordered AFM Heisenberg without the parity structure.
- Legacy work / regression checks: load the Z+ZZ family from the
  same directory. These fixtures are preserved unchanged.
- Cross-checking the parity break: see Part 4 of
  `scripts/scratch_qf_e4z_27_verification.jl` — it computes
  `‖[P, H]‖_∞` on both families. Z+ZZ shows 0.0 exactly; X+ZZ shows
  O(disorder_strength) = O(0.1).

**Caveat: not a re-run of qf-e4z.23.** Only the n=5 and n=7 seed=42
cells are generated by default (Part D of the qf-e4z.27 issue). Full
180-cell re-runs of qf-e4z.23 on the new X+ZZ fixtures are out of
scope; a separate follow-up (filed as a new beads issue if needed)
would do that.

**Generator entry point:** `run_1d_heisenberg_XZ_disorder()` in
`scripts/scratch_multiseed_disordered_fixtures.jl`. The default
`build_heis_1d` signature in `src/hamiltonian.jl` accepts
`disordering_terms=[[X], [Z, Z]]` directly — no `src/` changes needed
to add the new family.
